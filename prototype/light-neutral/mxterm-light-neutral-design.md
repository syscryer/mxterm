# mXterm 亮色中性主题设计说明

## 设计方向

这版设计按用户截图的 XTerminal 工作区结构收敛：顶部只放终端连接标签，左侧是连接仓库，中间是远程编辑器和多终端区域，右侧是可切换工具面板。界面使用亮色中性主题，不使用绿色，不做卡片式工作台。

## 必须保持

- 顶部连接标签只表示 SSH 连接，不承载文件标签。
- 新建 SSH 连接需要有独立界面，入口来自顶部连接标签的 `+`。
- 文件标签只出现在编辑器内部。
- 左侧窄图标栏去掉。
- 顶部最近连接、仓库密码等入口去掉。
- 区域之间贴边，只用细分隔线，不留大间隙。
- 左下角只保留设置按钮。
- 连接仓库顶部使用小图标入口，不显示大号文字标题。
- 上传、下载作为文件工具按钮保留。
- 工作区左侧固定放置连接仓库，文件、传输、监控等工具放在右侧可切换面板。
- 新增连接、编辑连接统一使用弹窗，不使用右侧固定表单。
- 连接仓库、主工作区、右侧面板之间的分隔线必须支持拖拽调整宽度。
- 编辑器和终端之间的分隔线必须支持拖拽调整高度。
- 右侧工具面板必须支持收起和展开，收起后保留窄按钮用于恢复。
- 右侧工具面板顶部只保留工具标签和必要的收起按钮，不放弹出、最大化、关闭等冗余窗口按钮。
- 密码和私钥口令输入框允许通过眼睛按钮查看，符合个人工具的直觉体验。

## 视觉原则

- 主背景：参考 codem 的冷白和浅灰体系，避免米黄、暖黄或绿色。
- 主文本：中性深灰，保证长时间使用可读性。
- 功能焦点色：浅灰胶囊用于选中状态和活动标签，主操作按钮才使用蓝色。
- 状态辅助色：监控和进度优先使用灰蓝，不使用黄色作为默认提示色，红色只用于危险或错误。
- 面板边界：使用 1px 极浅灰分隔线，避免厚边框和大圆角卡片。
- 信息密度：接近桌面运维工具，优先保证文件树、编辑器、终端和监控同时可见。

## 页面结构

- 顶部：应用标识、SSH 连接标签、新建连接、搜索和窗口按钮。
- 左一：紧凑连接仓库，顶部只放小图标，内容包含最近连接、收藏、分组和快速访问。
- 中间：文件编辑标签、编辑器工具栏、代码编辑区、底部多终端标签。
- 右侧：工具标签包含文件、搜索、传输、监控，默认显示文件工具栏、远程路径和文件树。
- 传输与监控：作为右侧工具面板内的可切换内容，不默认占用主工作区。
- 分隔线：连接仓库宽度、右侧面板宽度、终端高度都可拖拽。
- 左下角：单独的设置按钮。

## SSH 连接界面

- 顶部仍然是连接标签，新建连接显示为 `新建 SSH` 标签。
- 左侧是连接仓库分组：最近连接、收藏、开发环境、测试环境、生产跳板、云主机。
- 中间是 SSH 主机列表，保留协议切换、搜索、新建 SSH、连接、编辑操作。
- 新建 SSH 和编辑连接都打开弹窗，弹窗内包含名称、主机、端口、用户名、认证方式、密码或私钥路径、初始目录、编码、超时。
- 密码和私钥口令使用密码输入框，并提供查看按钮。
- 弹窗底部操作是取消、测试连接和连接，不出现额外的账号管理入口。

## 交互建议

- 图标按钮需要 tooltip 和 aria-label。
- 面板宽度后续支持拖拽调整，拖拽时只显示细线提示。
- 文件上传下载需要进入任务队列，并在右侧传输面板呈现进度。
- 文件区支持按钮上传和拖拽上传，拖到空白区域上传到当前目录，拖到目录行上传到该目录。
- 右侧文件路径栏只显示远程路径，不显示缩放或其他状态信息。
- 编辑器保存前检查远程文件修改时间，避免静默覆盖。
- 编辑区顶部先放文件标签，下面放撤销、重做、刷新、保存、下载、定位、缩放、搜索、换行符和编码工具条。
- 当前文件查找替换使用 Monaco 内置浮层，体验接近 VS Code。
- 远程目录搜索放在右侧搜索工具面板，不挤占当前文件查找浮层。
- 终端连接下可以打开多个终端标签，每个终端使用独立 SSH 会话，文件浏览和传输使用独立 SFTP/传输连接。

## 设置结构

- 左下角只放设置按钮。
- 设置分为终端、编辑器、文件传输、连接、外观和安全。
- 终端配色归入终端设置，不归入外观。
- 外观负责应用密度、字体大小、图标大小、面板尺寸记忆和启动时恢复上次布局。
- 终端配色支持内置多套方案，后续支持导入和导出。

## 错误提示

- 错误提示采用简短用户提示加原始详情展开的方式。
- 首版不做终端输入输出审计。
- 原始详情可以复制，但不能包含密码和私钥口令。
