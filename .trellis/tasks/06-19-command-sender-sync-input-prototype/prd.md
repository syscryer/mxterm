# Command Sender / Sync Input prototype

## Goal

在现有 mXterm light-neutral 原型中补一版 Command Sender / Sync Input 可点击原型，用来验证批量命令抽屉和同步输入持续状态的入口、目标选择、风险提示与反馈状态。

## Requirements

- 原型必须基于 `prototype/light-neutral/mxterm-empty-session.html` 继续迭代，不新起无关页面。
- 原型需要贴近现有 mXterm 会话结构：窗口标题栏显示首页、终端和 SSH 会话标签，工作区只保留当前 SSH 会话的子 tab 行。
- Command Sender 视觉上应是从终端底部打开的“命令操作台”抽屉，保持轻中性、高密度、桌面工具风格，不做孤立的大表单面板。
- SSH 子会话标签工具条右侧放置 Sync Input 文本开关和一个紧凑的 Command Sender 图标入口；Command Sender 不使用“开/关”大文字按钮占位。
- Command Sender 默认不常驻底部，不占用终端高度；点击子 tab 行右侧的图标入口后打开底部抽屉。
- Command Sender 的面板内关闭入口放在底部命令操作台标题栏，关闭后整块抽屉消失。
- Command Sender 底栏需要按会话展示已连接 SSH 终端目标，支持在目标区域内选择/取消目标，并提供全选控制；会话 tab 不承担批量目标选择。
- 当一个会话存在多个子 tab 时，目标行内提供子 tab 下拉选择；默认选择该会话当前活跃子 tab，而不是固定选择第一个子 tab。
- Command Sender 需要有命令输入区域、最近命令、追加回车选项、发送并回车、发送不回车、危险片段提示、发送结果，以及目标会话行内的投递状态小标签反馈。
- Command Sender 的发送按钮和结果反馈放在命令输入区底部，不单独占用右侧动作列，避免底部抽屉出现大块空白。
- Command Sender 的成功反馈只表示命令写入目标终端输入流，不判断远端命令执行结果。
- 目标会话行内的投递状态小标签需要可点击，点击后模拟切换到对应会话查看终端输出。
- Command Sender 是连续发送工具，发送后不自动关闭。
- Sync Input 需要在工具条上有独立显式开关，展示同步目标数量。
- 开启 Sync Input 后，终端区域需要有明显状态条，说明输入会同步到哪些目标。
- 原型只模拟交互，不调用真实后端、不发送真实终端输入。
- 首版不覆盖本地终端，不做命令历史、模板保存或权限系统。

## Acceptance Criteria

- [x] 打开原型后可看到 Sync Input 工具条入口和子 tab 行右侧的 Command Sender 图标入口，底部不默认常驻命令操作台。
- [x] 打开原型后可看到全局会话标签和当前 SSH 会话的子 tab 行，页面不额外增加第三层会话状态栏。
- [x] 点击 Command Sender 图标入口可打开底部命令操作台抽屉，点击标题栏关闭入口可关闭抽屉。
- [x] 勾选目标会更新目标数量、发送按钮状态和同步输入说明。
- [x] 目标区域的全选控制可一次选择或取消全部目标，并在部分选择时显示半选状态。
- [x] 多子 tab 会话可在目标行内切换投递子 tab，默认值为当前活跃子 tab。
- [x] 输入包含危险片段的命令时显示醒目的风险提示。
- [x] 发送按钮位于命令输入区下方，右侧不再保留独立空白动作列。
- [x] 连续点击发送会在原型中更新发送次数、目标会话行内投递小标签和发送结果，不触发真实终端写入。
- [x] 目标会话行内投递小标签可点击，并模拟切换到对应会话查看输出。
- [x] 开启 Sync Input 后终端区域显示同步输入状态条。
- [x] 关闭 Sync Input 后状态条消失。
- [x] 原型源码检查脚本通过。
