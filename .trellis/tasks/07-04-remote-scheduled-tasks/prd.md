# 远端定时任务工具

## Goal

在右侧“工具 > 定时任务”里维护当前 SSH 主机上的轻量定时命令，让用户不离开 mXterm 就能查看、创建、启停、删除和手动执行远端计划任务，并查看最近执行记录。

## Confirmed Facts

- 入口已经存在于 `DockerToolPanel` 的工具箱 tab，当前只显示“定时任务待接入”空状态。
- 工具面板已经复用当前 SSH 连接上下文，Docker 与网络诊断通过 `RemoteExecSessionPool` 执行远端命令。
- 用户授权按推荐方式实现，并计划明天验收。

## Requirements

- 使用远端系统能力作为执行来源，优先维护当前用户的 `crontab`，mXterm 关闭后任务仍应继续运行。
- 只管理 mXterm 自己创建的任务，不解析或改写用户手工维护的普通 crontab 行。
- 任务包含名称、cron 表达式、命令、启用状态、更新时间和最近执行摘要。
- 支持新增、编辑、启用/停用、删除、刷新和手动执行任务。
- 每次计划执行和手动执行都将输出追加到远端日志目录，面板可查看最近记录。
- UI 沿用现有远端工具箱紧凑面板风格、Lucide 图标、共享按钮/表单/状态样式和暗色主题 token。
- 不保存命令输出到本地，不记录敏感凭据；命令内容仅写入远端当前用户 crontab。

## Acceptance Criteria

- [ ] 点击“定时任务”tab 后显示任务列表、刷新入口和新增入口，不再是占位空状态。
- [ ] 无任务时显示可操作空状态；有任务时显示名称、启用状态、cron、命令摘要、最近执行结果。
- [ ] 新增/编辑任务会更新远端当前用户 crontab 中的 mXterm 管理块，不破坏管理块之外的 crontab 内容。
- [ ] 停用任务保留配置但不会被 cron 执行；启用后重新生效。
- [ ] 删除任务只移除对应 mXterm 管理项。
- [ ] 手动执行会立即通过 SSH 执行该任务命令，并刷新最近执行记录。
- [ ] 远端没有 `crontab` 命令、cron 表达式无效或命令为空时，前端展示可恢复错误。
- [ ] 运行项目约束检查和相关前端/后端测试；未要求完整编译则不执行完整编译。

## Out of Scope

- 不实现秒级调度、可视化 cron 生成器、复杂依赖编排、通知中心或跨主机批量任务。
- 不接管系统级 `/etc/crontab`、systemd timer、root 权限提权或其他用户的任务。
- 不实现长期日志归档；本次只展示最近若干条执行记录。
