# 统一终端和远程文件标签布局

## Goal

让远程文件编辑器可以按用户当前工作流从默认上下分屏切换到与终端共用顶部标签栏的统一 tab 布局，提升编辑器可视空间，同时保留现有默认分屏体验和终端稳定性。

## Requirements

- 当前 SSH 会话默认仍使用“远程文件编辑器在上、终端在下”的上下分屏布局。
- 用户可以把远程文件 tab 拖到终端 tab 栏，或把终端 tab 拖到远程文件 tab 栏，使当前 SSH 连接进入“统一 tab 模式”。
- 统一 tab 模式下，终端 tab 和远程文件 tab 共用同一个顶部 tab 栏；下方只显示当前激活 tab 对应的内容。
- 当前连接进入统一 tab 模式后，后续打开远程文件继续进入统一 tab 布局，不再自动恢复上下分屏。
- 统一 tab 模式下，用户可以把终端或文件 tab 拖回内容区，将当前连接恢复为上下分屏；也可以通过 tab 右键菜单执行“恢复上下分屏”。
- 关闭 SSH 会话或删除连接时，必须清理该连接的远程文件 tab 和统一布局运行时状态；重新打开同一连接不能自动恢复上次已关闭会话的文件 tab。若存在未保存文件，必须先确认放弃修改。
- 重新打开会话时按设置默认值决定布局。设置页新增“远程文件打开方式”，可选“上下分屏 / 统一 tab”，默认“上下分屏”。
- 第一版拖拽只负责切换布局模式，不做终端 tab 或文件 tab 的自由排序。
- tab 拖拽必须使用普通鼠标箭头样式，不使用 grab/grabbing 手型样式。
- 统一 tab 模式下，如果当前激活的是文件编辑器，顶部终端搜索和 Command Sender 按钮不展示；已打开的命令操作台应自动收起。
- 首次打开远程文件编辑器时，Monaco/编辑器 chunk 加载阶段必须展示与编辑器区域一致的紧凑 loading 状态，不能出现左上角裸文本。
- 统一 tab 模式必须保持终端实例挂载，不因为切换到文件 tab 而断开、重建或丢失缓冲。
- 变更必须沿用现有 WorkspaceShell、远程文件编辑器、设置页和全局样式体系，不引入新的视觉体系。

## Acceptance Criteria

- [ ] 默认设置下，打开远程文件仍显示上下分屏，终端 tab 栏和远程文件 tab 栏维持当前交互。
- [ ] 在上下分屏中把文件 tab 拖到终端 tab 栏后，当前连接切换为统一 tab，顶部同时显示终端和文件 tab。
- [ ] 在上下分屏中把终端 tab 拖到文件 tab 栏后，当前连接切换为统一 tab，顶部同时显示终端和文件 tab。
- [ ] 统一 tab 中点击终端 tab 只显示对应终端内容；点击文件 tab 只显示对应文件编辑器内容。
- [ ] 统一 tab 下打开新的远程文件后，新文件出现在统一 tab 栏并成为当前激活项。
- [ ] 统一 tab 下把任意终端或文件 tab 拖到内容区后，当前连接恢复为上下分屏。
- [ ] 统一 tab 下 tab 右键菜单提供“恢复上下分屏”，执行后恢复上下分屏。
- [ ] 关闭 SSH 会话后再打开同一连接，不展示上次会话已打开的远程文件 tab；如果会话内有未保存文件，关闭前先提示确认。
- [ ] tab 拖拽过程中鼠标仍为普通箭头，不显示 grab/grabbing 手型。
- [ ] 统一 tab 中激活文件 tab 时不显示终端搜索和 Command Sender 按钮，并收起已打开的命令操作台；切回终端 tab 后按钮恢复。
- [ ] 首次打开远程文件编辑器时，“正在加载编辑器”状态居中显示并使用现有 token 样式，不出现无样式裸文本。
- [ ] 切换到文件 tab 再切回终端后，终端连接、输出、搜索状态和输入响应不被重置。
- [ ] 设置页可修改“远程文件打开方式”，刷新后设置能被 normalize 并回显；默认值为“上下分屏”。
- [ ] 不影响本地终端、RDP、VNC、右侧文件面板、Command Sender 和远程文件保存/关闭/定位功能。
- [ ] 通过项目要求的启动边界检查、TypeScript/构建检查，以及远程文件编辑器相关 source check。

## Out Of Scope

- 第一版不做 tab 自由拖拽排序。
- 不做跨应用重启持久化“某个连接已切到统一 tab”的临时状态。
- 不把本地终端、RDP、VNC 或其他工具面板纳入统一 tab。
- 不改变远程文件读写、保存冲突、关闭确认、文件树定位等业务语义。
