# 连接配置优化与连接步骤

## Goal

把 SSH 连接配置从“保存后直接打开终端”升级为可长期维护的连接管理体验：连接配置清楚、凭据可复用、代理和少量高级项可配置，点击连接后先经过可观察的连接步骤，确认主机密钥和认证通过后才创建终端会话。

## Requirements

- 左侧连接仓库展示本地保存的 SSH 连接。
- 支持新增、编辑、删除连接，编辑使用弹窗，不在终端 tab 内编辑。
- 连接配置弹窗使用顶部页签，页签为“基本 / 代理 / 高级”，页签视觉应是文字页签加选中下划线，不做按钮形态。
- “基本”页签包含名称、分组、主机、端口、用户名、凭据来源、直接凭据字段和备注。
- 认证是高频配置，不单独作为页签。
- 分组是连接配置的一部分，连接应保存分组归属；左侧连接树从连接数据恢复分组，不只依赖浏览器 localStorage。
- 设置页新增“认证管理”或“凭据管理”分类，用于维护可复用 SSH 凭据。
- 凭据管理只保存凭据，不保存主机、端口、用户名。
- 凭据类型首版支持密码和私钥；私钥凭据包含私钥路径和可选私钥口令。
- 连接配置仍负责主机、端口和用户名；连接时组合主机、端口、用户名和凭据。
- 连接凭据来源支持使用保存的凭据、在此连接中保存、每次询问。
- 使用保存的凭据时，连接保存凭据引用；在此连接中保存时，连接保存直接凭据；每次询问时不保存密码或私钥口令。
- 代理页签本期支持不使用代理、HTTP CONNECT、SOCKS5，支持代理地址、端口和可选代理用户名密码。
- 不做代理链、多级代理、系统代理自动读取。
- 高级页签只展示本期真实实现的少量设置，优先包含连接超时、认证超时、心跳间隔。
- 终端编码只有在本期完成 SSH 输出和输入双向转码时才展示；如果不能完整实现，则本期不展示编码配置。
- “测试连接”和“打开终端”共用连接步骤页面，不直接把步骤日志写入终端。
- 点击连接后先进入连接步骤页面；主机密钥确认、临时凭据输入、认证、失败恢复都在步骤页面完成。
- SSH shell 成功准备好后才创建真正的终端 tab。
- 首次遇到主机密钥时必须展示确认动作，用户信任后记录到本机。
- 已信任主机密钥再次连接时不重复询问。
- 主机密钥变化时必须阻断连接，并给出明确的更新信任动作。
- 一个连接可以打开多个终端，每个终端拥有独立 SSH session、tab 标题和断开状态。
- 断开的 tab 保留历史输出和断开提示。
- 连接仓库区域在普通浏览器预览下可展示静态/内存状态，真实持久化和 SSH 连接只在 Tauri 环境启用。
- 未实现能力不提示、不展示；不要在界面里出现 RDP、VNC、Telnet、证书、FIDO2、Agent Forwarding、启动脚本、环境变量等尚未实现项。
- 密码、私钥口令和凭据首版仍可明文本地保存，后续锁屏密码或加密存储单独实现。
- 本任务不实现 SFTP、传输队列、远程编辑和软件锁屏。

## Acceptance Criteria

- [ ] Rust 后端提供 connection CRUD command，连接数据包含分组、凭据来源、代理配置和高级配置。
- [ ] Rust 后端提供 credential CRUD command，凭据数据只包含凭据相关字段。
- [ ] 连接配置和凭据配置都保存到本机应用数据目录，并能重新加载。
- [ ] 连接校验覆盖 host、port、username、分组、凭据来源、代理配置和高级配置。
- [ ] 凭据校验覆盖名称、类型、密码、私钥路径和私钥口令。
- [ ] 前端左侧连接仓库来自 connection command，并能按连接保存的分组恢复树结构。
- [ ] 新增和编辑连接通过顶部页签弹窗完成，页签为“基本 / 代理 / 高级”。
- [ ] 设置页提供凭据管理入口，可新增、编辑、删除密码和私钥凭据。
- [ ] 连接配置可选择保存的凭据，也可选择在连接中保存或每次询问。
- [ ] 删除连接和删除凭据前有确认；删除被引用凭据时有明确提示或阻断。
- [ ] 测试连接显示连接步骤页面，不创建终端 tab。
- [ ] 从连接仓库打开连接时显示连接步骤页面，成功后才创建终端 tab。
- [ ] 首次主机密钥确认不出现在终端内，而是在连接步骤页面完成。
- [ ] 主机密钥变化时阻断连接，并提供更新信任的明确动作。
- [ ] 代理连接至少覆盖 HTTP CONNECT 与 SOCKS5 的成功和失败路径。
- [ ] 同一个连接能打开多个终端 tab，tab 之间输出互不串线。
- [ ] 断开的 tab 保留历史输出和断开提示。
- [ ] 远程文件相关 SSH 连接复用同一套连接配置、凭据解析、代理和主机密钥信任逻辑。
- [ ] `pnpm check`、Rust 单测、`cargo check`、Trellis docs check 通过。

## Out of Scope

- RDP、VNC、Telnet 协议。
- 证书、FIDO2、Agent Forwarding、启动脚本、环境变量。
- 代理链、多级代理、系统代理自动读取。
- 软件锁屏和凭据加密迁移。
- SFTP、传输队列、远程编辑的新能力扩展。

## Notes

- 界面只展示本期真实实现的能力。
- 连接和凭据边界要保持清楚：连接负责目标和行为，凭据负责秘密材料。
- 连接步骤页面是测试连接和打开终端的共同入口。
