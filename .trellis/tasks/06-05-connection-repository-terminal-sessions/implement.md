# 连接配置优化与连接步骤实施清单

## Step 1：任务准备

- [ ] 确认当前工作区状态，识别已有未提交变更。
- [ ] 读取 `prd.md` 和 `design.md`。
- [ ] 运行 `trellis-before-dev` 并读取相关前端/后端规范。

## Step 2：Rust 数据模型与存储

- [ ] 扩展连接模型，加入分组、凭据来源、代理、高级配置。
- [ ] 扩展连接模型，加入 SSH 跳板机引用字段，首版只入库和校验，不接真实跳板链路。
- [ ] 为连接模型迁移旧数据写单测。
- [ ] 新增凭据模型和 `CredentialStore`。
- [ ] 为凭据新增、编辑、删除、校验写单测。
- [ ] 新增主机密钥信任模型和 `KnownHostStore`。
- [ ] 为未知、已信任、指纹变化三种 host key 状态写单测。
- [ ] 暴露 connection、credential、known host 相关 Tauri command。

## Step 3：统一连接解析

- [ ] 新增统一连接解析函数，把连接配置、凭据、代理、高级配置解析成运行时 SSH 请求。
- [ ] 终端连接改用统一解析函数。
- [ ] 远程文件连接改用统一解析函数。
- [ ] 测试连接改用统一解析函数。
- [ ] 删除被引用凭据时阻断或提示，避免连接引用悬空。

## Step 4：主机密钥确认

- [ ] 替换自动信任的 SSH handler。
- [ ] 在 `check_server_key` 中根据 known host store 判断通过、未知或变化。
- [ ] 将未知主机密钥返回为可恢复状态，包含算法和 SHA256 指纹。
- [ ] 将主机密钥变化返回为阻断状态，包含旧指纹和新指纹。
- [ ] 实现用户信任后写入或更新 known host store。
- [ ] 连接步骤确认后能重新执行连接并继续。

## Step 5：代理传输

- [ ] 实现无代理直连路径。
- [ ] 实现 HTTP CONNECT 代理 TCP 流。
- [ ] 实现 SOCKS5 代理 TCP 流，含可选用户名密码认证。
- [ ] 将代理后的 TCP 流交给 `russh::client::connect_stream`。
- [ ] 代理错误返回可读 AppError，并进入连接步骤日志。

## Step 6：前端类型与命令封装

- [ ] 更新连接类型，加入分组、凭据来源、代理、高级配置。
- [ ] 新增凭据类型和 credential command 封装。
- [ ] 新增连接步骤状态类型。
- [ ] 更新 demo/普通浏览器预览数据，保证无 Tauri 环境可浏览。

## Step 7：设置页凭据管理

- [ ] 设置页左侧导航新增“认证管理”或“凭据管理”。
- [ ] 实现凭据列表。
- [ ] 实现新增/编辑凭据弹窗或内联表单。
- [ ] 实现删除凭据确认和被引用提示。
- [ ] 连接弹窗中的凭据管理入口可跳转到该设置页。

## Step 8：连接编辑弹窗

- [ ] 将连接弹窗改为顶部页签样式，页签为“基本 / 网络路径 / 高级”。
- [ ] 基本页实现名称、分组、主机、端口、用户名、凭据来源、凭据选择、inline 凭据和备注。
- [ ] 网络路径页实现直连、HTTP CONNECT、SOCKS5、代理地址端口和可选认证。
- [ ] 网络路径页实现 SSH 跳板机入口，保存已保存连接引用，未选择跳板机时阻止保存和测试。
- [ ] 高级页实现连接超时、认证超时、心跳间隔和终端显示编码。
- [ ] 不展示未实现项。
- [ ] 保存时调用连接 upsert 并刷新连接树。

## Step 9：连接树分组迁移

- [ ] 从连接 profile 的分组字段构建连接树。
- [ ] 保留展开/折叠等纯 UI 偏好。
- [ ] 尝试迁移旧 localStorage 分组归属到连接 profile。
- [ ] 拖拽连接到分组时更新连接 profile，而不是只写 localStorage。

## Step 10：连接步骤页面

- [ ] 在主工作区新增连接步骤页面状态。
- [ ] 测试连接进入步骤页面，不创建终端 tab。
- [ ] 打开终端进入步骤页面，成功后才创建终端 tab。
- [ ] 实现步骤列表、日志、复制日志、编辑连接、取消、重试。
- [ ] 实现首次主机密钥确认动作。
- [ ] 实现主机密钥变化阻断和更新信任动作。
- [ ] 实现每次询问凭据的临时输入。
- [ ] 失败后保留步骤页面，不把错误写进终端。

## Step 11：终端与远程文件集成

- [ ] `TerminalPanel` 不再负责自动发起初始连接；由步骤页面成功后传入已准备好的 session 或触发 tab 创建。
- [ ] 多终端 tab 仍保持独立 session 和输出隔离。
- [ ] 终端 session 按连接高级配置解码 SSH 输出，并按同一编码写入终端输入。
- [ ] 远程文件面板使用统一连接解析后的 SSH 连接。
- [ ] 凭据、代理和 known host 逻辑在终端和远程文件路径表现一致。

## Step 12：验证

- [ ] 运行 Rust 单测。
- [ ] 运行 `cargo check --manifest-path src-tauri/Cargo.toml`。
- [ ] 运行 `pnpm check`。
- [ ] 运行 Trellis docs check。
- [ ] 手动验证普通浏览器预览。
- [ ] 手动验证 Tauri 窗口：保存连接、保存凭据、引用凭据、测试连接、首次信任、打开终端。
- [ ] 扫描敏感信息。
- [ ] 暂存新增代码和 Trellis 文件，不自动提交。

## Rollback Notes

- 数据模型改动必须保留旧连接数据迁移路径。
- known host store 初次接入后不能回退到自动信任。
- 如果代理实现风险过高，可优先保留无代理路径通过，但 UI 不能展示未完成的代理选项。
