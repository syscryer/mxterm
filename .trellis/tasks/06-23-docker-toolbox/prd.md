# Docker Toolbox

## Goal

在右侧工具区新增“工具”一级 tab，并在其中提供 Docker 管理能力。第一版实现 Docker 容器与镜像的日常管理；网络诊断和定时任务作为后续工具入口预留，但本轮不实现功能。

## Requirements

- 右侧一级工具入口增加“工具”，内部使用紧凑分段切换：`Docker / 网络诊断 / 定时任务`。
- Docker 页使用当前 SSH 连接在远端执行 `docker` CLI 和基础 systemd 命令；不连接 Docker TCP API，不直接访问 `/var/run/docker.sock`，不要求本机安装 Docker。
- 当当前工作区不是已连接 SSH 会话时，Docker 工具展示不可用状态，不应尝试执行命令。
- 当远端缺少 `docker`、当前用户无权限访问 Docker、或命令执行失败时，显示明确错误和原始原因。
- 容器视图展示容器名称、镜像、状态、端口、运行时间/创建时间等精简信息。
- 容器操作支持刷新、启动、停止、重启、删除、查看日志、进入容器终端、复制容器名/ID。
- 镜像视图展示 repository、tag、image id、大小、创建时间等精简信息。
- 镜像操作支持刷新、拉取镜像、删除镜像、复制镜像名/ID；拉取提交后在镜像列表内展示拉取中、当前层、进度和失败/完成状态。
- 引擎视图展示 Docker 服务状态、版本、Root Dir、Storage、Cgroup、容器/镜像/网络/卷数量、Daemon CPU/RAM 和磁盘占用。
- 引擎操作支持刷新、启动、停止、重启 Docker 服务；停止和重启必须确认。
- 引擎配置支持读取和编辑 `/etc/docker/daemon.json`，保存前做 JSON 校验，远端写回前自动备份原文件；“保存并重启”必须确认。
- 删除容器、删除镜像必须使用共享确认弹窗；第一版只做单项操作，不做批量删除。
- 容器日志第一版展示最近若干行，支持手动刷新；不做持续 tail 流式订阅。
- 进入容器终端应打开新的 SSH 终端 tab，并执行 `docker exec -it <container> sh` 或可用 shell；不在 Docker 面板内嵌完整终端。
- 网络诊断、定时任务页本轮仅显示轻占位和后续说明，不创建后端命令。
- 不做 sudo 密码交互；远端用户权限不足时直接展示错误。

## Acceptance Criteria

- [ ] 用户能在右侧工具栏打开“工具”tab，并在内部切换 Docker、网络诊断、定时任务。
- [ ] Docker 页能列出当前 SSH 连接远端的容器和镜像。
- [ ] 容器启动、停止、重启、删除、日志查看、进入终端动作可用，并有错误反馈。
- [ ] 镜像拉取、删除、复制动作可用，并有错误反馈；拉取期间不只停留在弹窗等待，镜像列表要展示进度行。
- [ ] 引擎页能展示 Docker 状态、基础信息和配置文件。
- [ ] Docker 服务停止、重启、保存并重启都有确认；权限不足时提示远端原始原因。
- [ ] 配置保存前校验 JSON，保存时远端自动备份原 `/etc/docker/daemon.json`。
- [ ] 删除类动作必须确认，取消确认不会执行远端命令。
- [ ] 远端缺少 Docker 或权限不足时，面板显示可理解的状态，不崩溃、不静默失败。
- [ ] 本轮不修改已有终端、文件、监控、隧道和命令功能的行为。
- [ ] UI 使用现有 Radix/Lucide/共享样式和 `--mx-*` token，不引入独立视觉体系。

## Out of Scope

- 不做 Docker Compose / Stack 管理。
- 不做创建容器、编辑容器配置、镜像 build、镜像 push、volume/network 操作管理。
- 不做 prune、一键清理或批量危险操作。
- 不做实时 stats 图表或长期历史记录。
- 不做网络诊断和定时任务的真实执行逻辑。
- 不做 sudo 密码输入、sudoers 配置或非 systemd 主机的服务控制兼容。
