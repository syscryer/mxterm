# Integrate transfer panel into file pane

## Goal

将右侧“传输”从一级工具 tab 调整为文件面板内的底部传输状态条和可展开队列，降低右侧工具栏拥挤度，同时保持现有传输功能与状态流完全不变。

## Requirements

- 不修改后端传输命令、事件、取消、重试、冲突策略、目录压缩传输、打开/定位本地路径等功能逻辑。
- 右侧一级工具入口移除“传输”，保留“文件 / 监控 / 隧道 / 命令”。
- 文件 tab 内底部显示传输状态条，包含传输标题、进行中数量、失败数量、整体进度和展开/收起入口。
- 展开后显示传输队列，保留现有任务操作：取消、重试、移除、复制路径、打开、定位、清理完成项。
- 传输队列常态列表只展示精简信息：文件类型图标、文件名、方向、状态点、进度、大小、速度、状态和关键操作。
- 传输队列运行态状态文案应优先表达当前阶段，例如目录压缩传输展示“压缩中”，逐文件目录下载展示“下载中/扫描中”，不要一律显示“进行中”。
- 传输失败时保留失败前的最后进度和大小信息，不要为了强调失败把进度伪装成 100%。
- 长路径、速度、阶段、错误原因等详细信息通过 tooltip/hover 详情展示，避免常态列表拥挤。
- 空传输状态应保持安静，不影响文件树浏览。
- 样式必须使用现有 `--mx-*` token 和全局工具面板风格，不照搬截图样式，不新增独立视觉体系。

## Acceptance Criteria

- [ ] 右侧一级工具栏不再出现“传输”tab。
- [ ] 文件面板底部能看到传输状态条；有任务时可展开查看队列。
- [ ] 传输队列仍能触发取消、重试、移除、复制路径、打开、定位、清理完成项。
- [ ] `RemoteFileTransferPanel` 的数据结构和上层回调保持兼容，不改 Tauri 传输命令。
- [ ] 顶部状态条在右侧窄栏宽度下不发生文本重叠。
- [ ] 任务行使用文件类型图标，方向通过标签展示。
- [ ] Hover 详情不遮挡常态文本，不使用硬编码主题色。
- [ ] 原型文件保留为体验参考。

## Notes

- 这是 UI 迁移，不是传输功能重构。
