# 隧道入口迁移到工具

## Goal

把隧道从当前会话右侧工具面板的一级标签中移出，归入低频「工具」能力中。隧道仍然按全局后台网络服务理解，不绑定某个终端 tab 的生命周期；本次只调整入口和展示归属，不重写隧道运行模型。

## Requirements

- 右侧会话面板不再出现独立的「隧道」一级 tab。
- 「工具」入口中应能访问隧道能力，用户仍可查看和管理隧道规则。
- 现有 Docker 工具入口不能被破坏；如果工具入口需要承载多个工具，应保持现有紧凑桌面面板风格。
- 迁移后不改变后端隧道命令、数据结构或运行语义。
- UI 继续使用现有 Radix/Lucide/共享组件与 `--mx-*` token，不引入独立视觉体系。
- 不影响 AI 面板、命令库、监控、远程文件等已有右侧面板入口。

## Acceptance Criteria

- [x] 会话右侧工具栏不再显示「隧道」一级标签。
- [x] 用户可以通过「工具」访问隧道管理界面。
- [x] Docker 工具仍可从「工具」访问，且不和隧道界面互相覆盖。
- [x] 本地终端入口保持高频直达，不被降级为子 tab。
- [x] `npm run check` 通过。
- [x] `node scripts/check-startup-module-boundary-source.mjs` 通过。

## Notes

- 本次不新增网络诊断、端口占用、代理等工具，只为后续工具集合保留清晰入口。
