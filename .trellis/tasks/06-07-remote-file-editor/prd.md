# 完整远程文件编辑器

## Goal

为 mXterm 接入完整远程文件编辑能力，让用户可以在 SSH 会话旁边直接浏览、打开、编辑、保存和管理远程服务器文件。编辑体验采用 Monaco Editor，整体形态接近桌面 SSH 工具里的“终端 + 远程文件 + 编辑器”工作区，而不是只做一个临时文本预览框。

用户价值：连接到远程服务器后，可以从右侧文件树双击文件，在中间工作区用 Monaco 编辑、保存、处理冲突，并完成常见文件管理操作，减少在终端里反复 `vim/scp` 的切换成本。

## Confirmed Facts

- 当前 `RemoteFilePanel` 已支持按连接列出远程目录、路径跳转、刷新、显示隐藏文件、定位到当前终端目录和目录展开。
- 当前后端 `RemoteFileManager` 已复用 SSH exec session 执行目录 listing，但还没有读文件、写文件、创建、重命名、删除、上传或下载命令。
- 当前中间工作区已经有终端 session tab 和 terminal stack，需要扩展为“终端 tab + 远程文件编辑 tab”的统一工作区，并保持 SSH 终端不被卸载。
- 用户明确要求完整实现，避免后续返工；编辑器选择 Monaco Editor，而不是 textarea 或 CodeMirror。
- 项目 UI 规范要求前端保持桌面工具感，复用 Radix + Lucide + 现有 CSS token，不引入新的整套 UI 框架。
- 当前上传入口只是临时本地文件 input，下载入口只是浏览器 Blob 下载；还没有 Tauri 桌面级本地文件选择、默认下载目录、拖拽上传或配置化下载策略。
- 当前文件树右键菜单已有打开、下载、重命名、删除等基础入口，但还需要补齐复制路径、复制文件名/目录名、复制父目录等常用操作。
- 文件夹上传和下载需要走 `tar.gz` 归档链路，避免逐文件传输带来大量 SSH 往返和状态管理复杂度。

## Requirements

- 使用 Monaco Editor 作为远程文件编辑器核心。
- 右侧远程文件树中，文件可双击打开；目录点击保持展开/收起行为。
- 打开的远程文件在中间工作区以 tab 形式显示，和终端 tab 同属主工作区。
- 文件编辑区应作为当前终端区域上方的插入面板出现；文件 tab 只属于编辑面板，终端 tab 保持原位置，避免形成两层全局 tab，也避免打开文件后替代或卸载当前终端。
- 文件编辑器头部应保持紧凑：文件名已经在 tab 中出现时，不再额外展示大标题；路径、保存状态和工具按钮合并到一行，整体高度接近工具图标高度。
- 文件编辑区和终端区之间应支持垂直拖拽调整高度，并保持终端与 Monaco 状态不卸载。
- 终端区域使用暗色终端主题时，终端 tab 条应使用普通浅色面板样式；只有亮色终端主题才使用玻璃效果。
- 文件 tab 至少展示文件名、dirty 标记、关闭按钮；活动文件区展示连接名、远程路径、保存状态。
- 支持多文件打开；重复打开同一连接同一路径时切到已有 tab，不重复创建。
- 支持编辑器工具栏：
  - 保存。
  - 重新加载。
  - 放弃更改。
  - 查找/替换入口，优先使用 Monaco 内置查找能力。
  - 关闭文件。
- 支持快捷键：
  - `Ctrl+S` / `Cmd+S` 保存当前文件。
  - Monaco 默认撤销/重做、查找等快捷键不应被外层全局快捷键破坏。
- 支持 Monaco 基础能力：
  - 行号。
  - 语法高亮。
  - 选中态和光标。
  - 自动布局。
  - 跟随设置里的终端/等宽字体或至少使用统一 `--font-mono` 字体栈。
- 支持文本文件类型识别：
  - shell、ini/conf、json、yaml、toml、markdown、js/ts、python、rust、css/html、nginx 配置等常见语言。
  - 无法识别时作为纯文本。
- 远程读文件：
  - 新增 typed Tauri wrapper，不让组件直接调用 `invoke(...)`。
  - 返回内容、路径、文件名、大小、mtime、mode、是否二进制、是否可编辑等元数据。
  - 大文件默认不直接编辑，建议阈值 2 MB；超过阈值提示下载或使用终端工具处理。
  - 二进制文件不进入 Monaco 编辑，提示不可编辑。
- 远程写文件：
  - 新增 typed Tauri wrapper。
  - 保存前检查打开时的远端 mtime/size 或等价版本信息。
  - 若远端文件在打开后变化，提示用户选择重新加载、覆盖保存或取消。
  - 写入应尽量使用临时文件 + 原子替换，避免写一半损坏原文件。
  - 写入内容不能直接拼进 shell 命令字符串；必须避免特殊字符、换行、引号和命令长度导致保存失败。
- 文件管理操作：
  - 新建文件。
  - 新建文件夹。
  - 重命名只修改当前条目的名称，不允许在重命名弹窗里直接改父目录路径；父目录需要作为只读上下文展示。
  - 删除文件或文件夹，删除前确认。
  - 右键菜单按桌面文件管理器体验组织，常用动作放一级，包含刷新、上传文件、上传文件夹、新建文件、新建文件夹、下载、打开方式、重命名、复制绝对路径、查看属性、终端、其他和删除等动作。
  - 复制路径只复制远程绝对路径，不再拆出复制名称、父目录或 scp 路径等低频变体。
  - 查看属性作为一级菜单项，展示权限、大小、修改时间、类型、远程绝对路径等信息。
  - 打开方式菜单至少支持“内置编辑器打开”和“下载后用系统默认程序打开”；对目录不显示文件打开方式。
  - 终端菜单至少支持“在此处打开终端 / 复制 cd 命令 / 定位当前终端目录”等远程终端联动操作。
  - 其他菜单承载低频操作，例如复制远程 scp 路径、复制相对路径、查看属性/权限信息、刷新父目录。
  - 顶部上传文件：上传到当前路径，至少支持单文件；优先支持多文件批量上传。
  - 文件面板工具栏行的上传按钮必须完整接入上传文件和上传文件夹，使用当前路径作为目标目录，并和右键菜单/拖拽上传复用同一套上传逻辑与冲突处理。
  - 目录右键上传文件：上传到对应目录。
  - 拖拽上传：本地文件拖到目录行时上传到该目录，拖到当前文件列表空白区时上传到当前路径。
  - 上传文件夹：先把本地文件夹打成 `tar.gz`，上传到远端临时归档文件，再在目标目录解压，完成后清理远端临时文件。
  - 下载文件夹：远端先把目录打成 `tar.gz` 临时归档，下载到本地默认下载分组目录，再解压为文件夹，完成后清理远端临时归档。
  - 文件/文件夹上传或下载遇到同名目标时不静默覆盖，提供覆盖、跳过、重命名上传/下载的确认路径。
  - 下载文件：默认保存到用户下载目录下的会话分组目录，例如 `Downloads/<会话名称>/202606080123/`。
  - 下载目录策略本轮在设置中配置，包含“默认下载根目录 / 是否按会话分组 / 时间戳目录格式 / 是否保留压缩包 / 默认冲突策略”。
- 下载体验：
  - 右键下载和文件操作下载默认使用本地下载目录，不再只依赖浏览器 Blob 下载。
  - 拖拽下载：从远程文件树拖动文件或目录时触发下载到当前下载策略指定目录；下载完成后可打开或定位本地结果。
  - OS 级拖出到 Windows 资源管理器/桌面按完整体验调研并实现；若 Tauri 官方跨平台 API 不支持，采用 Windows-first 原生实现或明确的平台能力说明，同时保留可用的默认目录下载链路。
  - 目录下载默认解压到下载分组目录，并支持在设置中选择是否保留压缩包。
  - 下载完成后给出明确反馈，并尽量展示本地保存路径。
- 传输体验：
  - 上传/下载需要有任务状态反馈，至少展示进行中、成功、失败和本地/远端目标路径。
  - 多文件和文件夹传输需要逐项记录结果，失败项不应阻塞已成功项展示。
  - 支持取消尚未开始的队列项；正在执行的 SSH 传输如无法安全中断，需要明确展示“本项完成后停止”。
  - 传输状态作为右侧工具区中与“文件”同级的“传输”面板展示；文件面板或工具栏可显示传输数量徽标并作为入口，不使用长期遮挡编辑区的浮窗。
- 文件树在写入、创建、重命名、删除、上传后刷新相关目录，并尽量保留展开状态。
- 关闭 dirty 文件时必须确认，避免误丢修改。
- 断开连接、切换连接或文件面板重载时，已打开编辑 tab 的 dirty 状态不能静默丢失。
- 浏览器预览环境不应发起真实 SSH；可以使用 mock/preview 数据展示编辑器状态。
- 不把编辑器塞进右侧窄文件面板；右侧保持浏览和操作入口，中间负责编辑。
- 不在本轮实现 Git diff、远端 LSP、远端终端联动保存命令、sudo 保存或完整 IDE 项目管理。

## Acceptance Criteria

- [ ] 文件树中文件可双击打开为中间 Monaco 编辑 tab。
- [ ] 打开多个文件时，tab 切换、关闭、dirty 标记和重复打开去重正常。
- [ ] Monaco 编辑器可编辑文本文件，显示行号，自动适配容器尺寸，并使用 mXterm 的等宽字体风格。
- [ ] `Ctrl+S` / `Cmd+S` 能保存当前活动文件。
- [ ] 保存成功后 dirty 标记清除，相关远程目录刷新。
- [ ] 保存失败时显示明确错误，不丢失编辑内容。
- [ ] 远端文件打开后被修改时，保存会进入冲突提示：重新加载、覆盖保存、取消。
- [ ] 关闭 dirty 文件时出现确认，不会静默丢弃修改。
- [ ] 二进制文件和超过大小限制的文件不会直接进入编辑器，并给出明确提示。
- [ ] 新建文件、新建文件夹、重命名、删除、下载、上传文件和上传文件夹入口可用。
- [ ] 重命名弹窗只编辑被选中文件/目录的名称，提交后在原父目录下生成新路径，不会因为用户误改完整路径而移动到其他目录。
- [ ] 上传文件夹使用 `tar.gz` 归档上传并在远端目标目录解压；下载文件夹使用远端 `tar.gz` 归档下载并在本地默认目录解压。
- [ ] 上传/下载遇到同名文件或文件夹时提供覆盖、跳过、重命名的确认策略，不静默覆盖。
- [ ] 文件树右键菜单支持复制远程绝对路径，并使用系统剪贴板能力或可用降级方案。
- [ ] 文件、目录和空白区右键菜单按不同上下文展示合适动作；上传文件、新建文件、下载、重命名、复制绝对路径、查看属性等常用动作位于一级菜单。
- [ ] 文件面板工具栏上传、目录右键上传、拖拽上传可把本地文件或文件夹上传到正确远程目录；上传后刷新相关目录。
- [ ] 下载文件和目录默认保存到用户下载目录下的 `<会话名称>/<yyyyMMddHHmm>/` 目录，保存路径策略可在设置页配置。
- [ ] 拖拽下载可从文件树发起下载到当前下载策略目录；OS 级拖出下载按 Windows-first 能力实现或给出明确平台限制说明。
- [ ] 传输状态反馈覆盖上传、下载、文件夹归档/解压、成功、失败和取消。
- [ ] 右侧工具区包含“文件 / 传输”同级面板；传输面板展示队列、阶段、进度、失败、取消、打开目录和复制路径。
- [ ] 所有新增 Tauri 命令都有 `src/shared/tauri/commands.ts` typed wrapper。
- [ ] 后端文件保存不通过把完整内容拼进 shell 命令字符串实现。
- [ ] `npm run check`、`npm run build` 通过；后端命令相关 Rust 测试通过或说明既有阻塞。
- [ ] 浏览器/本地应用验证覆盖：打开文件、编辑、保存、关闭 dirty 文件、二进制/大文件拦截、文件树刷新。

## Notes

- 编辑器选择：Monaco Editor。
- 当前优先保证文本编辑、保存安全、文件树管理和完整上传下载链路；远端 LSP、sudo 保存、Git 级 diff 不进入本轮。
