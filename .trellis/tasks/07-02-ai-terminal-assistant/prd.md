# AI 终端小帮手

## Goal

在 mXterm 内提供一个面向终端排障和命令生成的 AI 对话小帮手，帮助用户解释终端输出、分析错误原因、生成或改写命令，并把结果安全地带回现有命令操作台或命令库。

第一版按“完整一期”规划，不只做轻量聊天壳；需要覆盖模型配置、上下文管理、终端排障对话、命令建议、命令落地、安全提示和基础会话体验。不实现自动执行型 Agent 模式。用户已有自己的 AI Agent 能力，后续可作为独立阶段接入。

## Confirmed Facts

- 当前项目已有右侧工具面板体系，`RemoteFileTool` 包含 `files`、`monitor`、`tunnels`、`commands`、`tools`。
- 当前项目已有命令库与命令历史能力，命令库右侧面板能复制、插入、发送命令；底部 Command Sender 是现有目标选择和终端写入入口。
- 当前项目已有 MCP 设置入口，用于把 mXterm 保存连接提供给本机 AI Agent，并包含 SSH 操作开关和危险命令控制。
- 本任务第一版不是 Agent 模式，不要求 AI 自动调用 SSH 工具、远程文件工具、Docker 工具或 MCP 工具。
- AI 对话应遵守现有 mXterm 右侧面板风格、全局 token、暗色主题和启动性能约束。
- AI 对话第一版入口确定放在右侧工具面板一级「AI」标签中。后续隧道工具可能迁出当前右侧工具面板，因此第一版不需要因为现有标签数量过度压缩 AI 入口。
- 模型接入第一版支持两个 provider：OpenAI 和 Claude。用户自研 Agent 接入作为后续扩展，不在本阶段实现。
- 模型认证第一版只支持 API Key；不实现 OAuth、网页登录、账号授权或浏览器跳转登录。
- 模型配置采用配置列表，每条配置至少包含：配置名称、接入模式、API Key、请求地址、模型。这里的“配置名称”就是用户口径里的供应商/配置名；界面只展示接入模式，内部仍由接入模式派生后端需要的 `provider` 与 `api_format`。用户可以保存多条配置，并在 AI 对话面板中切换当前使用的配置。
- 终端写入已有 `terminalWrite` 和底部 Command Sender 路径；命令库已有复制、插入、发送、保存片段等相关能力，可作为 AI 命令建议落地的集成点。
- 现有 `TerminalPanel` 已能通过 xterm selection API 读取选中文本；文件面板和标签页已使用 Radix ContextMenu 与全局 `context-menu-*` 样式，可作为终端右键菜单的实现风格参考。
- 上下文策略确定采用可视化上下文包：默认支持终端选中文本、最近终端输出、当前连接脱敏信息、命令草稿/最近命令等上下文块，发送前展示来源、范围、字符量或行数，并允许用户移除。
- AI 对话历史需要本地持久化，直接保留完整对话内容，包括用户消息、AI 回复以及当次发送的可见上下文块。
- AI 回复必须支持流式输出，并提供停止生成能力。
- AI 对话输入遵循常见聊天习惯：默认 `Enter` 发送，`Shift+Enter` 换行；发送快捷键归入现有「快捷键」设置页统一管理。
- 验证阶段不使用 Codex 自带浏览器工具；若需要交互验证，使用命令行、构建、类型检查、自动化测试或非浏览器方式。

## Requirements

- 提供一个内置 AI 对话入口，定位为“终端排障和命令生成小帮手”。
- 入口放在右侧工具面板的一级「AI」标签，与命令库、监控、工具箱保持同一类桌面辅助工具体验。
- AI 能围绕用户提供的终端上下文回答问题，重点支持：
  - 解释报错、异常输出、命令失败原因。
  - 根据用户意图生成 shell 命令。
  - 改写或补全已有命令。
  - 给出排查步骤和风险提示。
- AI 生成的命令默认只作为建议展示，用户可复制、插入到命令操作台、或保存为命令片段。
- 第一版不允许 AI 未经用户确认直接执行命令。
- 第一版不实现 Agent 模式，不接入自动工具调用闭环；后续接入应保留扩展空间。
- 第一版模型配置必须支持 OpenAI Chat Completions 与 Claude Messages 两种接入模式，配置项包含配置名称、接入模式、API Key、请求地址、模型。
- 请求地址必须可配置，以支持官方 API、国内代理、企业网关或兼容服务。
- 接入模式必须可配置，至少支持 OpenAI Chat Completions 兼容模式和 Claude Messages 原生模式，避免把配置名称和实际请求协议强绑定。
- AI 对话面板必须支持从已保存的 AI 配置列表中切换当前模型配置。
- API Key 必须按现有安全存储思路保存，不应以明文长期保存在前端状态、日志或普通配置文本中。
- 第一版必须具备完整可用的对话体验，包括加载中、错误、重试、停止生成、空状态、配置缺失引导、会话清空或新会话入口。
- AI 对话输入框默认按 `Enter` 发送消息，`Shift+Enter` 保留为换行；用户可在「快捷键」设置中调整或禁用发送快捷键。
- AI 回复必须流式展示；用户能在生成过程中停止输出，停止后保留已生成内容并标记状态。
- AI 对话历史必须本地持久化保存完整对话，支持会话列表、继续对话、删除单个会话、清空当前会话。
- 第一版必须具备完整上下文体验：用户能看到将发送给 AI 的上下文来源和范围，并能控制是否附加终端选中文本、最近终端输出、当前连接的脱敏信息、命令草稿/最近命令等终端排障相关上下文。
- 终端选中文本必须支持右键菜单入口：用户在终端中用鼠标选中文字后，可通过右键菜单打开 AI 面板，并将选中内容发送到 AI 对话框作为可见上下文。
- 右键发送选中文本后不得自动提交给模型；AI 面板应等待用户补充问题并明确发送。
- 右键菜单入口应使用项目现有菜单样式与交互能力，避免为终端单独创建不一致的菜单视觉体系。
- 第一版必须把 AI 生成的命令作为结构化建议呈现，支持复制、插入到命令操作台、保存为命令片段；如支持发送到终端，也必须显式确认，不能静默执行。
- 第一版需要基础安全体验：对明显危险命令、凭据/密钥/token 风险、批量删除/格式化/权限提升等高风险建议给出清晰提示。
- 普通命令建议不需要二次确认；只有用户选择将危险命令发送到终端时才需要二次确认。复制、插入到命令操作台、保存为片段不应被确认弹窗打断。
- 发送给 AI 的上下文必须可控，敏感信息不得默认发送。
- AI 功能配置应能与后续用户自研 Agent 接入方案兼容，避免把当前实现锁死为单一供应商或单一路由。

## Acceptance Criteria

- [ ] 用户能从 mXterm 的合适入口打开 AI 终端小帮手。
- [ ] 右侧工具面板包含一级「AI」入口；该入口不依赖后续隧道迁移才能正常使用。
- [ ] 用户能输入问题并获得 AI 回复。
- [ ] AI 回复以流式方式显示，用户能停止生成并保留已生成内容。
- [ ] AI 对话支持加载中、错误、重试、停止生成、清空/新会话、配置缺失引导等完整状态。
- [ ] AI 对话输入框默认 `Enter` 发送、`Shift+Enter` 换行，且发送快捷键出现在「快捷键」设置页。
- [ ] AI 对话历史本地持久化保存完整对话，用户能查看历史会话、继续对话、删除单个会话、清空当前会话。
- [ ] 用户能把终端相关上下文带入对话，且上下文来源、范围、字符量或行数对用户可见。
- [ ] 用户选中终端文本后，右键菜单提供发送到 AI 对话的入口，打开 AI 面板，并将选中内容作为可见上下文附加到对话框。
- [ ] 右键发送选中文本不会自动提交模型请求，必须等用户点击发送或使用等价明确动作。
- [ ] AI 回复中的命令不会自动执行。
- [ ] 对命令建议提供复制、插入到现有命令操作台、保存为命令片段的路径；任何直接发送到终端的能力都必须显式确认。
- [ ] 高风险命令建议会显示危险提示，不把风险隐藏在普通文本回复里。
- [ ] 普通命令发送到终端不需要二次确认；危险命令发送到终端前必须二次确认。
- [ ] Agent 模式明确不在本阶段实现，界面或设置不暗示已经具备自动执行能力。
- [ ] 用户能配置并选择 OpenAI 或 Claude 作为 AI 对话模型 provider。
- [ ] OpenAI 与 Claude 均使用 API Key 认证；不要求用户通过 OAuth 或网页登录授权。
- [ ] 用户能维护多条 AI 配置，每条配置包含配置名称、接入模式、API Key、请求地址、模型，并能在 AI 对话面板中切换。
- [ ] UI 符合现有右侧面板、共享组件、全局 token、亮暗主题约束。
- [ ] 功能不增加首屏静态重模块；涉及启动路径时必须通过启动模块边界检查。
- [ ] 验证不依赖 Codex 自带浏览器工具。

## Out Of Scope

- 自动执行命令、自动修复、自动调用远程文件/Docker/隧道/监控工具。
- 本任务不移动现有隧道工具入口；隧道迁出是后续独立工作。
- 多 Agent 编排、工具调用规划器、长期任务模式。
- 训练或微调模型。
- 将终端密钥、密码、token、完整连接配置默认发送给 AI。
