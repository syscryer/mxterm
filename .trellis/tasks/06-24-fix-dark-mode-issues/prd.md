# 修复暗色模式视觉问题

## Goal

集中修复 mXterm 暗色模式下的视觉断层，让工作区、设置页、监控面板和命令操作台在深色主题中保持统一的桌面工具风格。

用户价值：开启深色主题后，界面不再出现大面积浅色卡片、浅色图表、浅色输入框或低对比文字，远程监控和批量命令发送等高频工作面板仍然清晰可读。

## Confirmed Facts

- 用户提供的截图显示，深色主题已能应用到标题栏、左侧连接树、终端和部分右侧监控容器。
- 截图中的右侧监控面板存在明显浅色残留：磁盘读写 tile、磁盘挂载卡片、网络信息卡片、上下行 tile 和流量图表背景仍接近白色。
- 截图中的设置外观页左侧导航呈浅色半透明/低对比状态，右侧设置内容虽然较暗，但需要保持与全局 token 体系一致。
- 截图中的命令操作台大体已是深色，但需要检查底部操作区、目标卡片、输入框、按钮禁用态和选中态是否仍有浅色或低对比问题。
- `src/features/monitor/monitor.css` 中存在浅色硬编码背景，例如 `#fbfcfe`、`#e8ebf3`、`#eef1f8`、浅色渐变和浅色告警背景，这些是监控面板深色主题不一致的直接来源。
- `src/styles/app.css` 中设置页、预览区、命令操作台和若干全局设置控件仍有浅色硬编码或 light-only 视觉值。
- 项目规范要求前端视觉必须优先使用 `src/styles/tokens.css` 中的 `--mx-*` token，并通过共享样式落地；不要在 feature 里散落一次性浅色/暗色补丁。
- 当前仓库已有未提交改动：`README.md`、`.mcp.json`、`docs/assets/`，本任务不能回滚或覆盖这些无关改动。

## Requirements

- 优先修复截图可见范围：
  - 右侧监控面板的磁盘、网络、流量图表、状态卡片、输入/按钮/告警等子组件。
  - 设置外观页的导航、内容背景、预览区、设置行、分段控件、输入和按钮状态。
  - 命令操作台的目标列表、输入区、底部按钮、禁用态、选中态和关闭按钮。
- 暗色主题下不能出现大块纯白或接近白色的业务 surface，除非它是明确的内容预览且有足够边界和语义说明。
- 浅色主题现有观感不能被明显破坏；修复应通过 token、`color-mix()` 或 `.app-shell[data-theme-mode="dark"]` / system-dark 对应规则实现。
- `data-theme-mode="system"` 且系统为深色时，应与显式 `data-theme-mode="dark"` 使用同等的修复规则。
- 监控图表在深色主题下要保留数据可读性：网格线、折线、面积填充和背景对比要清楚但克制。
- 按钮、输入框、选择框、checkbox、状态 badge、禁用态、错误/警告态在深色主题下要有清晰边界和足够文字对比。
- 不借由隐藏组件、删除信息、关闭图表或把数据改透明来掩盖视觉问题。
- 不引入新的 UI 框架、图标库或独立配色系统。
- 不自动提交 git；若新增代码类文件需要暂存，但本任务预计只改现有文件和 Trellis 文档。

## Acceptance Criteria

- [ ] 深色主题工作区右侧监控面板不再出现截图中的浅色磁盘/网络 tile 和浅色流量图表底板。
- [ ] 深色主题监控面板所有主要文本、次级文本、图标、边框、进度条和图表线条保持可读。
- [ ] 深色主题设置外观页左侧导航和右侧内容区与 mXterm 面板风格一致，设置控件状态清晰。
- [ ] 深色主题命令操作台目标选择、输入框、底部操作按钮、禁用态和选中态没有低对比或浅色断层。
- [ ] 浅色主题仍保持现有轻量桌面工具风格，没有被暗色修复带偏成过深或过重。
- [ ] `system` 主题在系统深色偏好下应用同一组暗色修复。
- [ ] 样式改动优先复用 `--mx-*` token 和共享 CSS，不在业务组件中新增硬编码暗色体系。
- [ ] 运行项目既有前端校验命令；如未编译，应说明是遵守“未要求编译时不要编译”的项目规则。

## Out Of Scope

- 不重做暗色主题的整体视觉语言。
- 不重新设计监控面板信息架构或数据采集逻辑。
- 不修复截图之外的所有历史浅色硬编码，除非它们与本轮可见面板共享同一规则且顺手消除风险更低。
- 不调整终端配色方案本身；终端 surface 只保持现有 xterm 主题。
