# 改进远程文件本地图标

## Goal

提升远程文件面板中文件/文件夹图标的清晰度、丰富度和离线可靠性，让常见目录与文件类型在无网络环境下也能显示合适的本地图标。

## Requirements

- 远程文件面板图标不得依赖网络 CDN 或外部在线资源加载。
- 图标视觉尺寸需要比当前 16px 更大，接近参考截图里的文件树图标大小。
- 默认文件夹图标需要更明亮，避免在浅色界面里显得偏黑/偏脏。
- 常见目录和文件类型应有更丰富的区分，至少覆盖 Dockerfile、docker compose、配置文件、脚本、压缩包、安装包、证书/密钥、README/LICENSE，以及常见 Linux/开发目录。
- 保持远程文件数据模型和远程文件命令不变，只调整前端图标解析与渲染。
- 图标应保持紧凑，不破坏文件树行高、缩进、点击、展开和右键菜单行为。

## Acceptance Criteria

- [x] 远程文件图标不再引用 `https://cdn.jsdelivr.net` 或其他网络 URL。
- [x] 文件树图标在视觉上大于当前 16px，约 20px 级别。
- [x] Dockerfile 能显示 Docker 类型图标。
- [x] 普通文件夹图标比当前默认文件夹更亮，展开状态仍可区分。
- [x] 常见文件类型和目录名获得比默认文件/文件夹更丰富的图标或颜色。
- [x] TypeScript 类型检查通过。

## Notes

- 用户明确要求：不要网络加载图标；图标要大一些，接近截图中对比产品的大小。
- 当前实现通过 `remoteFileIcons.ts` 返回 vscode-icons CDN URL，`RemoteFilePanel` 用 `<img>` 渲染并在失败时 fallback。
- 推荐实现为本地 React/SVG 图标组件和类型解析，不新增在线依赖。
