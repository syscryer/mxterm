# Credential security model

## Goal

重新设计并落地 mXterm 的连接内置 secret、账号管理凭据和高级安全保护模型。默认面向个人用户保持无打扰；开启高级保护后，用一个总安全密码控制 vault 解锁、闲置锁定和安全设置修改。

## Requirements

- 默认普通模式下不弹安全密码，不限制查看密码；连接内置密码和账号管理凭据可通过眼睛按钮查看。
- 底层 secret 仍必须加密保存到 `secrets.enc`，不得明文写入 SQLite、日志或前端持久化。
- 连接编辑页只处理当前连接的内置密码和内置私钥口令；引用账号管理凭据时，只显示凭据引用，不跨层展示公共凭据 secret。
- 连接编辑保存时，未触碰内置 secret 字段必须保留旧 secret；输入新值才替换；移除 secret 必须是明确动作或认证方式切换，不能靠空输入误删。
- 账号管理是公共凭据 secret 的唯一查看、替换和删除入口。
- 高级保护关闭时，vault 使用本机 local key 自动解锁，普通用户无感。
- 高级保护开启时，vault 使用总安全密码加密；应用启动后必须输入总安全密码才能解锁 secret。
- 高级保护开启后，安全设置页默认锁定；输入总安全密码后可一次性编辑全部安全配置。
- 高级保护开启后的配置项只保留：
  - 闲置自动锁定：不自动、5、15、30、60 分钟。
  - 允许查看已保存密码：开启或关闭。
  - 修改安全密码。
  - 关闭高级保护。
- 关闭“允许查看已保存密码”后，连接编辑和账号管理不展示眼睛按钮；用户仍可替换 secret。
- 闲置锁定只清除内存中的 vault 解锁状态和 secret 读取能力，不主动断开已有 SSH 会话。
- 开启、关闭高级保护和修改安全密码都属于 vault rekey，必须保证原子写入，失败时不得丢失原 secret。

## Acceptance Criteria

- [ ] 连接编辑已有内置密码时，打开编辑页不明文回显；不修改保存后仍能用旧密码连接。
- [ ] 连接编辑已有内置密码时，点击眼睛可临时显示明文；输入新密码保存后替换旧值。
- [ ] 引用账号管理凭据的连接编辑页不展示公共凭据 secret，只保留凭据选择和账号管理入口。
- [ ] 账号管理凭据支持查看和替换已保存密码/私钥口令；未修改保存时保留旧 secret。
- [ ] 设置页高级保护总开关关闭时，不展示高级安全子选项。
- [ ] 开启高级保护后，安全设置页默认锁定；解锁后展示并可修改允许查看密码、自动锁定时间、修改安全密码、关闭高级保护。
- [ ] 高级保护开启且允许查看密码关闭时，连接编辑和账号管理都不展示眼睛按钮。
- [ ] 高级保护 rekey 操作有后端测试覆盖，失败不破坏现有 vault。
- [ ] 后端 reveal/preserve 行为有测试覆盖。
- [ ] 前端类型检查通过，关键 Rust 测试通过。

## Out of Scope

- 不做每个连接或每个凭据单独的安全密码。
- 不做每次点击眼睛都要求输入安全密码。
- 不把 WebDAV 同步密码纳入本次 UI 设计；WebDAV 保持现有保存和同步主密码逻辑。
- 不实现自动断开 SSH 会话。
