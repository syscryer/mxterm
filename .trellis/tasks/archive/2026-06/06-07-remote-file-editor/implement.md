# 完整远程文件编辑器实现计划

## Steps

1. 依赖和验证护栏
   - 添加 Monaco Editor 依赖。
   - 新增源码检查脚本，覆盖 Monaco 接入、typed Tauri wrappers、远端保存不拼接完整内容、dirty/冲突 UI 入口。
   - 确认 Vite + Monaco worker 配置方式。

2. 后端远程文件契约
   - 在 `commands.rs` 新增 read/write/create/rename/delete/download/upload request/response。
   - 在 `remote_files.rs` 新增元数据读取、文本读取、二进制/大小拦截。
   - 为 `ReusableExecSession` 新增带 stdin 的 exec 方法。
   - 实现保存：版本检查、stdin 写临时文件、原子替换、返回新 metadata。
   - 为 shell quoting、metadata parsing、二进制检测、冲突检测、保存命令构造添加 Rust 单元测试。

3. 前端 typed wrappers 和类型
   - 扩展 `src/shared/tauri/commands.ts`。
   - 新增 `RemoteFileReadResult`、`RemoteFileEditorTab`、`RemoteFileSaveState` 等类型。
   - 新增路径到 Monaco language 的映射工具。

4. 工作区 tab 模型
   - 将中间工作区 tab 扩展为 terminal/file union。
   - 文件双击打开时创建或激活文件 tab。
   - 文件 tab 支持 dirty 标记、关闭按钮、关闭确认。
   - 保持终端会话不卸载。

5. Monaco 编辑器组件
   - 新增 `RemoteFileEditor`。
   - 动态加载 Monaco，创建/dispose model。
   - 接入保存、重新加载、放弃更改、查找、关闭。
   - 接入 `Ctrl+S` / `Cmd+S`。
   - 使用 mXterm 等宽字体和紧凑工具栏样式。

6. 文件树操作补全
   - 文件双击打开。
   - 文件菜单：刷新、上传文件、下载、打开方式、重命名、复制绝对路径、查看属性、终端、其他、删除。
   - 目录菜单：刷新、上传文件、上传文件夹、新建文件、新建文件夹、下载目录、重命名、复制绝对路径、查看属性、终端、其他、删除。
   - 文件树空白区菜单：刷新当前目录、上传文件、上传文件夹、新建文件、新建文件夹、下载当前目录、复制当前绝对路径、终端。
   - 打开方式子菜单：内置编辑器打开、下载后用系统默认程序打开、以纯文本打开。
   - 复制路径动作只复制远程绝对路径，不做名称、父目录、scp 路径二级菜单。
   - 查看属性弹窗/面板展示权限、大小、修改时间、类型、远程绝对路径。
   - 修正重命名弹窗：只编辑当前条目的 basename，父目录只读展示；提交时用原父目录 + 新名称生成目标路径。
   - 重命名名称校验：不能为空，不能包含 `/` 或 `\`，不能是 `.` / `..`。
   - 终端子菜单：在此处/父目录打开终端、复制 cd 命令、定位当前终端目录。
   - 其他子菜单：刷新父目录等低频动作。
   - 操作后刷新相关目录并保留展开状态。

7. 上传和下载产品化
   - 接入 Tauri 本地文件选择/保存能力，补齐 package 依赖、capability 权限和 typed helper。
   - 文件面板工具栏上传菜单包含上传文件、上传文件夹，目标目录为当前路径。
   - 目录右键上传文件上传到该目录；所有上传入口复用同一套传输调度、冲突确认和状态反馈。
   - 上传文件夹：本地打 `tar.gz`，上传到远端临时归档，远端解压到目标目录，清理临时文件。
   - 下载文件默认保存到用户下载目录下的 `<连接名称>/<yyyyMMddHHmm>/`。
   - 下载文件夹：远端打 `tar.gz`，下载到本地临时目录，本地解压到下载分组目录，清理临时文件。
   - 支持拖拽本地文件或文件夹到目录行/当前文件列表空白区上传，并显示 drop target 高亮。
   - 支持拖拽远程文件或目录发起下载到当前下载策略目录；Windows 平台调研并接入 OS 级拖出下载到资源管理器/桌面。
   - 上传/下载同名目标时按“覆盖 / 跳过 / 重命名”确认策略处理，避免静默覆盖。
   - 上传后刷新目标目录；下载后展示本地保存路径并提供打开/定位。
   - 设置页接入下载根目录、按连接分组、时间戳目录格式、是否保留压缩包、默认冲突策略。
   - 建立传输状态反馈，覆盖文件/文件夹上传下载、归档、解压、成功、失败、取消。
   - 右侧工具区增加“文件 / 传输”同级 tab；文件 tab 保持文件树，传输 tab 展示队列。
   - 传输 tab 徽标显示活动/失败任务数量；从文件面板发起传输时不强制切换，失败时高亮提示。
   - 传输队列项支持取消未开始任务、显示本地/远端路径、复制路径、下载完成后打开/定位本地目录。

8. 冲突、错误和边界状态
   - 保存冲突弹窗：重新加载、覆盖保存、取消。
   - 二进制/超大文件拦截页。
   - 读取失败/保存失败状态。
   - 连接丢失或连接配置删除后的 tab 状态。
   - 上传/下载失败保留当前目录状态，并显示可定位的错误信息。
   - 归档/解压失败显示阶段信息：本地打包、远端上传、远端解压、远端打包、下载、本地解压或清理。

9. 验证
   - `node scripts/check-remote-file-editor-source.mjs`
   - 增加或更新源码检查脚本覆盖复制路径菜单、拖拽上传入口和下载目录策略。
   - 增加或更新源码检查脚本覆盖文件夹 `tar.gz` 上传/下载、拖拽下载和传输状态。
   - `npm run check`
   - `npm run build`
   - `npm test`
   - `cargo test` 或 `npm run tauri` 相关可用命令中的 Rust 测试。
   - 浏览器验证：打开文件、编辑、dirty 关闭确认、保存状态、二进制/大文件拦截、文件树刷新、右键复制菜单、拖拽上传/下载视觉状态。
   - Tauri 验证：顶部上传、目录右键上传、拖拽上传、文件夹上传、右键下载、目录下载、拖拽下载、默认保存目录、下载完成反馈、设置项生效。

## Risk Points

- Monaco 体积较大，必须动态加载或合理配置 worker，避免拖慢首屏。
- 保存内容不能拼进 shell 命令；必须用 stdin 或等价安全传输方式。
- 远端 `stat` 输出在不同系统上可能差异较大；命令应尽量使用 POSIX 常见能力并有解析测试。
- 原子替换可能改变 owner/mode；本轮至少保留 mode，owner 保留失败不应阻塞普通用户保存。
- 上传/下载如果走本地文件 picker，需要 Tauri dialog/fs 能力；若当前插件未接，需要分阶段接入或明确先实现命令层。
- 下载默认目录包含连接名称，需要统一清理路径非法字符；连接名称优先，不使用终端 tab 标题。
- 拖拽上传容易和连接树拖拽样式/事件冲突；文件树只处理 `DataTransfer.files`，不要误吃连接分组拖拽。
- 文件夹上传/下载依赖本地归档能力和远端 `tar`/`gzip`；缺失时要返回明确错误，不能静默退化成部分传输。
- OS 级拖出下载可能需要 Windows 原生能力；如果 Tauri 官方 API 不足，需要隔离平台实现并提供稳定默认目录下载路径。
- 当前工作区已有大量 WIP，修改 `RemoteFilePanel`、`WorkspaceShell`、`app.css` 时要避免覆盖无关改动。

## Review Gate

实现前确认：

- Monaco 作为编辑器核心。
- 本轮大小限制默认 2 MB。
- 文件夹上传/下载必须完整接入，使用 `tar.gz` 归档链路。
- sudo 保存不进入本轮。
- OS 级拖出下载做 Windows-first 实现或明确平台能力限制；下载必须有默认目录、右键、菜单和拖拽入口。
- 下载默认路径使用 `<用户下载目录>/<连接名称>/<yyyyMMddHHmm>/`，设置页必须可配置。
