# 修复终端导航与 SSH 会话激活态混淆

## Goal

修复顶部“终端”入口处于激活状态时，左侧 SSH 连接列表项也同时呈现激活态的问题，让主导航激活态和连接会话选中态表达不同层级的状态。

## Requirements

- 顶部“终端”入口表示当前主工作区处于本地终端/终端工作区视图。
- 左侧 SSH 连接列表项只在对应 SSH 会话是当前活动远程会话时呈现选中/激活态。
- 进入本地终端视图时，左侧 SSH 连接项不应因为最近打开过或存在会话而显示为激活态。
- 修复应从状态来源或条件判断上解决，不用纯样式遮盖来隐藏错误状态。
- 保持现有连接点击、会话切换、终端 tab 和首页导航行为不回退。

## Acceptance Criteria

- [ ] 当前处于顶部“终端”主导航时，“终端”入口显示激活态。
- [ ] 当前处于顶部“终端”主导航时，左侧 SSH 会话/连接项不显示激活态，除非用户明确切回该 SSH 会话。
- [ ] 点击左侧 SSH 连接进入远程终端时，该连接项仍可显示当前远程会话的选中态。
- [ ] TypeScript 类型检查通过。

## Notes

- 用户截图指出：顶部“终端”是激活状态，旁边/侧边的 SSH 会话不应同时是激活状态。
- 这是轻量 UI 状态修复，PRD-only 足够；不需要单独 design.md / implement.md。
