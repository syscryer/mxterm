# Command snippets and history

## Goal

在现有 Command Sender 基础上补“命令片段”和“历史命令”能力，让用户可以快速复用常用运维命令、从最近发送记录回填命令，并把有价值的历史命令沉淀为长期片段。

## User Value

- 常用命令不需要反复手打或从外部笔记复制。
- 连续运维时可以快速回填最近发送过的命令。
- 片段和历史与 Command Sender 的多目标投递能力共用，不新增一套独立发送工具。
- 命令片段和历史可写入 SSH 终端，也可写入本地终端。
- 默认历史只来自用户通过 Command Sender 主动发送的内容；用户显式开启后，才记录保守过滤后的终端回车输入。

## Requirements

- 右侧工具面板增加“命令”tab，tab 内分“片段 / 历史”两个视图；底部 Command Sender 仍是唯一发送操作台。
- 命令片段使用一层树形文件夹分组，文件夹标题下直接缩进展示片段；支持列表、复制、回填、执行、保存当前输入为片段、编辑、删除和使用次数统计。
- 片段字段至少包含标题、命令内容、文件夹分组、可选说明、标签、收藏状态、使用次数、最后使用时间、创建/更新时间。
- 历史命令在发送后自动记录，按最后使用时间倒序展示，重复命令合并并累加使用次数。
- 历史命令支持按范围筛选：当前 SSH 连接、其它 SSH 连接、本地终端 profile（PowerShell/cmd/Git Bash 等）和全部历史；SSH 工作区默认当前连接，本地终端工作区默认当前 profile。
- 历史命令以紧凑命令行列表展示，支持回填、执行、保存为片段、删除单条、清空全部。
- 复制片段或历史命令只复制文本，不记录历史，也不增加片段使用次数。
- 历史命令默认只记录 Command Sender 成功写入至少一个目标后的命令；不记录空命令、不记录目标 session id 或完整目标清单。
- “记录终端输入”开关放在系统设置中，默认关闭；开启后仅记录用户在 SSH 或本地终端中按回车提交的普通可打印命令行。历史视图只展示当前状态和设置入口。
- 终端输入历史遇到控制序列、方向键、Tab 补全、Ctrl/Alt 类输入、疑似密码命令或敏感变量时必须丢弃当前行，不做强行兜底记录。
- 历史来源需要区分主动发送和终端输入，UI 上用紧凑标签标识。
- 本地终端工作区右侧工具面板只显示“命令”工具，隐藏文件、传输、监控、隧道等 SSH 专属工具。
- 发送后仍清空输入框，保持现有连续发送模型。
- 如果用户选择片段后直接发送，片段使用次数和最后使用时间应更新；如果选择后又修改命令，按普通历史命令处理。
- 从右侧片段/历史点击直接发送时，不自动展开底部 Command Sender，也不清空底部 Command Sender 中正在编辑的内容。
- 从右侧片段/历史直接发送时，目标列表可包含 SSH 终端和本地终端；写入成功才记录历史或增加片段使用次数。
- 数据持久化使用 SQLite，通过 Rust Tauri commands 暴露给前端；不要使用 localStorage 作为生产存储。
- Browser preview 或无 Tauri runtime 时可以展示空状态或轻量假数据，但真实操作必须通过 typed wrapper。
- UI 必须贴合现有右侧工具面板和 Command Sender 的紧凑桌面工具风格，使用 Lucide、Radix、共享按钮/菜单样式和全局 `--mx-*` token。
- 本轮不做普通终端被动历史采集、不做跨设备同步、不做片段变量模板、不做权限审批。

## Acceptance Criteria

- [ ] 右侧工具面板能切换到“命令”tab，并在 tab 内切换“片段 / 历史”。
- [ ] 片段按文件夹分组展示，选择命令片段可以把命令写入 Command Sender 编辑区。
- [ ] 选择历史命令可以把命令写入 Command Sender 编辑区。
- [ ] 当前输入可以保存为命令片段，保存后立即出现在片段列表。
- [ ] 用户可以为片段设置文件夹分组，空分组直接展示在根目录，不生成“未分组”文件夹。
- [ ] 用户可以编辑和删除已保存片段。
- [ ] 用户可以复制片段和历史命令，复制不会写入历史或增加使用次数。
- [ ] 发送成功后历史命令持久化，关闭再打开应用仍可看到最近历史。
- [ ] 重复发送同一条命令不会产生重复历史行，而是更新最近使用时间和使用次数。
- [ ] 历史命令默认按当前 SSH 连接或当前本地 profile 筛选，用户可切换到其它连接/profile 或全部历史。
- [ ] 历史命令支持删除单条和清空全部。
- [ ] 从片段直接发送后，片段使用次数增加，最后使用时间更新。
- [ ] 从右侧片段/历史直接发送不会弹出底部 Command Sender。
- [ ] 默认情况下普通终端键盘输入不会写入历史命令。
- [ ] 设置中开启“记录终端输入”后，普通回车命令会进入对应连接/profile 历史，控制序列/疑似敏感输入不会进入历史。
- [ ] 本地终端页右侧只显示“命令”工具，片段/历史可以发送到本地终端。
- [ ] Tauri command wrappers、Rust command 签名和前端类型一致。
- [ ] `npm run check` 通过；后端新增单元测试覆盖片段校验、历史合并、删除/清空和列表排序。

## Out of Scope

- 不做 shell hook、shell history 文件解析、exit code 解析或命令耗时统计。
- 不做片段变量、占位符、参数弹窗或模板渲染。
- 不做命令执行结果解析；发送状态仍只表示写入终端输入流。
- 不把命令片段加入 WebDAV 同步范围，后续如果要同步再扩展 `SyncDataDocument`。
- 不做团队共享、审批、审计日志或权限系统。

## Notes

- 现有 Command Sender 已有内存态 `commandSenderHistory`，本任务会迁移为后端持久化历史。
- 现有 Command Sender 面板仍是唯一发送入口；右侧命令 tab 只是辅助回填、触发和管理能力。
