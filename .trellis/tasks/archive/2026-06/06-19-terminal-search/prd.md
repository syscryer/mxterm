# 终端搜索

## Goal

为 SSH 终端和本地终端补一个轻量的输出搜索能力，让用户可以在当前终端缓冲区里查找文本并上下跳转匹配项。

## Requirements

- 搜索入口放在终端子 tab 右侧工具区，使用紧凑 Lucide 图标按钮。
- 点击搜索入口后在当前终端区域顶部显示紧凑搜索条，不常驻占用明显空间。
- 搜索条支持输入关键字、上一个、下一个、关闭。
- 搜索能力必须接入 xterm 官方搜索能力，匹配结果高亮在终端缓冲区内，不用自行解析 DOM。
- SSH 终端和本地终端都应支持搜索；搜索状态按终端 tab 独立保存。
- 搜索为空时跳转按钮禁用；关闭搜索条时清空当前高亮并把焦点还给终端。
- 搜索 UI 使用现有 token、Lucide 图标、Tooltip 和紧凑工具条风格，不引入新的 UI 框架。
- 不做全局跨会话搜索、搜索历史、正则搜索、大小写开关或结果总数统计。

## Acceptance Criteria

- [x] 子 tab 工具条右侧出现终端搜索图标入口，并与现有 Command Sender / 右侧面板按钮并排。
- [x] 点击搜索入口后，当前活动终端显示搜索条，其他终端不被强制打开搜索条。
- [x] 输入关键字后可在当前终端缓冲区查找并高亮匹配项。
- [x] 上一个、下一个按钮能调用当前终端的向前/向后搜索。
- [x] 关闭搜索条会清除搜索高亮并恢复终端焦点。
- [x] SSH 终端和本地终端都能使用同一套搜索能力。
- [x] 搜索 UI 不使用原生下拉或新样式体系，遵守全局 token。
- [x] 源码检查和 TypeScript 检查通过，或明确记录既有阻塞。
