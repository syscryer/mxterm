# 实现右侧文件管理面板

## Goal

实现轻量右侧文件管理面板，让活动 SSH 会话具备可浏览的远程目录树，并能手动定位到当前终端所在目录，减少手动输入路径和逐层展开目录的成本。

## Requirements

- 右侧工具面板首版只做文件管理，不做远程搜索标签；搜索能力暂不进入本任务范围。
- 文件管理面板只在存在活动 SSH 会话时展示，并绑定当前活动连接/终端。
- 面板需要包含远程路径栏、刷新入口、树形目录/文件列表和紧凑文件类型小图标。
- 文件树视觉参考 `D:\project\codem`：目录/文件使用类似 VS Code 的小图标，行高紧凑，目录优先，支持展开/折叠。
- `cd` 定位方向是“终端 -> 文件树”：用户点击定位按钮时，右侧文件树只使用当前激活终端 tab 已记录到的目录并刷新到该目录。
- 不做终端目录变化后的自动文件树刷新，避免每次 `cd` 都触发远程目录读取。
- 不做“文件树目录 -> 一键 cd 当前终端”作为首要能力，避免把用户意图反向。
- 首版不允许文件管理向交互终端写入命令或探测当前目录；终端目录只能来自已记录的 `OSC 7` 信号或用户实际输入的 `cd` 命令，未知时定位按钮禁用/提示。
- 上传、下载、传输队列、文件内容搜索和监控面板不属于本任务范围，可后续单独做。

## Acceptance Criteria

- [ ] 右侧面板不再出现“搜索”工具标签。
- [ ] 活动 SSH 会话下右侧默认展示文件管理面板；无活动会话时不强行展示文件工具。
- [ ] 文件树展示目录和文件，小图标随文件/目录类型变化，并有加载失败 fallback。
- [ ] 文件树目录可展开/折叠，路径栏能显示当前远程目录。
- [ ] 用户在终端输入 `cd /path` 后，点击定位按钮能让文件管理面板定位或刷新到该目录；目录未知时不执行命令、不影响终端。
- [ ] 文件管理实现不把所有逻辑堆在 `WorkspaceShell`，应拆到文件管理相关组件/工具中。
- [ ] `npm run check` 和 `npm run build` 通过，或说明无法通过的既有原因。
