# 修复连接信息拖拽动画

## Goal

恢复左侧连接列表中“连接信息”条目的拖拽动画反馈，让用户拖动连接时能看到清晰、连续的拖拽状态，而不是静默移动或完全没有动效。

## Requirements

- 定位拖拽动画消失的根因，优先从连接列表/分组列表的 DnD 状态、样式和最近改动中排查。
- 修复时保留现有连接列表样式、选中态、分组结构和点击连接行为。
- 不用列表去重、隐藏元素或兜底样式掩盖根因；应在拖拽事件/状态/样式源头恢复动画。
- 保持前端 UI 风格统一，优先复用现有组件和样式约定。
- 附带微调 xterm 内容左侧留白，避免终端文字紧贴工作区边缘，同时不影响 FitAddon 尺寸测量。

## Acceptance Criteria

- [ ] 左侧连接区域内拖动连接信息时，有可见的拖拽中/悬停反馈或过渡动画。
- [ ] 连接条目仍可正常点击打开终端，分组展开/收起和选中态不回退。
- [ ] 终端正文左侧至少保留 3px 视觉留白，xterm 鼠标选择/输入坐标不偏移。
- [ ] 修复不影响右侧文件管理面板、终端页签和连接错误提示 UI。
- [ ] 通过项目可用的前端质量检查，或明确记录无法运行的原因。

## Notes

- 这是轻量 bugfix，PRD-only 即可。
