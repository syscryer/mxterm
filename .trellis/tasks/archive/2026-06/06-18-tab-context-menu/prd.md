# 标签右键菜单

## Goal

为会话标签、终端子会话标签和远程编辑器标签补充类似 VSCode 的自定义右键菜单，让标签关闭、批量关闭和编辑器路径复制操作更顺手。

## Requirements

- 顶部会话标签需要支持自定义右键菜单，菜单项包括关闭、关闭其他、关闭右侧标签页、全部关闭。
- SSH 终端子会话标签需要支持自定义右键菜单，菜单项包括关闭、关闭其他、关闭右侧标签页、全部关闭。
- 本地终端子会话标签需要支持自定义右键菜单，菜单项包括关闭、关闭其他、关闭右侧标签页、全部关闭。
- 远程编辑器文件标签需要支持自定义右键菜单，菜单项包括关闭、关闭其他、关闭右侧标签页、关闭已保存、全部关闭和复制路径。
- 编辑器标签关闭未保存文件时必须继续走现有未保存确认流程，不得静默丢弃修改。
- 关闭已保存只关闭未修改的远程编辑器标签，保留 dirty、保存中、加载中、冲突或错误状态标签。
- 菜单 UI 必须复用现有 Radix ContextMenu 和共享 `.context-menu-*` 样式，并补充类似 VSCode 的右侧快捷键/说明列样式。
- 菜单项不可用时应显示 disabled 状态，不触发操作。
- 不新增真实快捷键绑定，本轮只展示快捷键/说明文本。

## Acceptance Criteria

- [x] 右键顶部会话标签时显示自定义菜单，关闭类操作作用于当前会话标签集合。
- [x] 右键 SSH 终端子会话标签时显示自定义菜单，关闭类操作作用于当前连接下的子会话标签集合。
- [x] 右键本地终端子会话标签时显示自定义菜单，关闭类操作作用于本地终端标签集合。
- [x] 右键远程编辑器文件标签时显示自定义菜单，关闭类操作作用于当前连接下的编辑器标签集合。
- [x] 远程编辑器标签的复制路径会把 `tab.path` 写入剪贴板。
- [x] 关闭未保存编辑器标签仍会触发现有确认弹窗。
- [x] 右键菜单视觉风格与现有远程文件树右键菜单一致，并有右侧快捷键/说明列。
- [x] `npm run check` 通过。

## Notes

- 用户参考 VSCode 标签右键菜单截图，希望“会话 - 子会话 - 编辑标签”都有类似右键菜单。
