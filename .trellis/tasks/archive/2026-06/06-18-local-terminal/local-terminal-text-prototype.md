# 本地终端文字原型

## 设计目标

本地终端作为顶栏一级入口存在，心智上与“首页”同级，而不是 SSH 连接树中的一种连接。用户进入本地终端后，在工作区内部通过子 tab 管理多个本机 Shell 会话。

## 顶栏结构

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ [侧栏] [首页] [本地终端]   [SSH: 203.0.113.70 ×] [SSH: K8s-node2 ×]     [_][□][×] │
└──────────────────────────────────────────────────────────────────────────────┘
```

- “首页”保留现有位置和行为。
- “本地终端”放在顶栏，与首页同级，推荐固定入口，不显示关闭按钮。
- SSH 会话仍然以可关闭的顶栏 tab 展示。
- 点击“本地终端”：
  - 如果没有任何本地终端子 tab，自动创建默认 profile 的子 tab。
  - 如果已有本地终端子 tab，恢复最近激活的本地终端子 tab。
- 默认 profile 初始为 PowerShell，设置中可修改。

## 本地终端工作区

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ PowerShell 7 ×   Windows PowerShell ×   cmd ×   [+] [⌄]                      │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  PowerShell 7.5.0                                                             │
│  PS D:\ai_proj\mXterm\m-xterm>                                                │
│                                                                              │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 子 tab 行只在“本地终端”顶栏标签激活时显示。
- 每个子 tab 对应一个真实本地 PTY 会话。
- 子 tab 标题默认使用 profile 名称，例如 `PowerShell 7`、`cmd`、`Ubuntu`、`Git Bash`。
- 子 tab 可关闭。关闭时如果会话仍在运行，需要确认，避免误杀本地进程。
- 本地终端工作区不显示 SSH 专属右侧文件、监控、传输工具，默认给终端完整宽度。

## 新建终端操作

```text
[+]     直接新建默认 profile
[⌄]     打开 profile 菜单
```

下拉菜单示例：

```text
新建本地终端
────────────────
● PowerShell 7        默认
  Windows PowerShell
  Command Prompt
  Ubuntu              WSL
  Debian              WSL
  Git Bash
  MSYS2
  Cygwin
  Cmder
────────────────
  管理 profiles...
  本地终端设置...
```

- `+` 是高频路径，始终创建默认 profile。
- 下拉用于选择具体 Shell 类型。
- 默认 profile 用圆点或轻量标识提示。
- 不可用 profile 不出现在菜单中。
- 自动发现 profile 和用户自定义 profile 后续统一进入这个菜单。

## 空状态

用户第一次点击顶栏“本地终端”时，不展示空引导页，直接创建默认 profile 子 tab。

如果默认 profile 启动失败：

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ PowerShell 7  启动失败 ×                         [+] [⌄]                     │
├──────────────────────────────────────────────────────────────────────────────┤
│ 无法启动 PowerShell 7                                                        │
│ D:\Program Files\PowerShell\7\pwsh.exe 不存在或无法访问                       │
│                                                                              │
│ [选择其他终端] [打开本地终端设置]                                             │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 失败状态保留在子 tab 内，不打断顶栏。
- 用户可以直接用下拉选择其他 profile。
- 设置入口用于修改默认 profile 或修正自定义命令路径。

## 设置入口

```text
设置
├─ 基础设置
├─ 凭据管理
├─ 外观
├─ 终端主题
└─ 本地终端
   ├─ 默认 profile: PowerShell 7
   ├─ 自动发现 profile
   │  ├─ PowerShell 7
   │  ├─ Windows PowerShell
   │  ├─ Command Prompt
   │  ├─ Ubuntu
   │  └─ Git Bash
   └─ 自定义 profile
      └─ 新增 profile
```

首版设置项建议：

- 默认 profile。
- 是否隐藏某个自动发现 profile。
- 自定义 profile：名称、命令路径、参数、启动目录、环境变量。

管理员 Cmd / PowerShell 首版只保留模型扩展点，不进入菜单。

## 会话与 SSH 的关系

- 顶栏层级：
  - 首页：固定入口。
  - 本地终端：固定入口，内部有本地子 tab。
  - SSH 会话：连接级顶栏 tab，可关闭。
- 左侧连接树：
  - 继续只放 SSH 连接、分组、连接管理。
  - 不放 PowerShell、cmd、WSL 等本地 profile。
- 右侧工具：
  - SSH 会话显示文件、监控、传输等远程工具。
  - 本地终端首版不显示右侧工具，也不新增本地专属边栏，默认保持终端全宽。

## 推荐交互细节

- 顶栏“本地终端”使用终端图标和文字，激活态与首页一致。
- 子 tab 行沿用现有终端 tab 风格，避免再做一套标签视觉。
- 不同类型会话在左侧使用不同小图标，例如 SSH、PowerShell、cmd、WSL、Git Bash 视觉上可快速区分。
- `+` 和下拉是相邻但分离的两个按钮：一个默认动作，一个选择动作。
- 当只剩最后一个本地子 tab 且用户关闭它时，保留“本地终端”顶栏入口，工作区回到轻量空状态或下次点击自动新建默认 profile。
- 本地终端输出、输入、resize、复制粘贴、搜索和主题沿用现有 `TerminalPanel` 能力。
