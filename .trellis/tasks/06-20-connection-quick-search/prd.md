# 连接快速搜索弹层

## Goal

为连接仓库增加一个参考 codem 的中央快速搜索弹层，让用户不需要在左侧连接树中逐级展开分组，也能快速按关键字找到并打开 SSH 连接。

## Confirmed Facts

- 当前左侧连接树位于 `src/features/connections/ConnectionPane.tsx`，已包含固定分组、用户分组、连接项右键菜单、收藏和刷新/新增入口。
- 首页连接仓库 `ConnectionHome` 已有内联搜索，但只在首页表格内使用，不覆盖左侧连接树或已打开会话时的快速切换场景。
- codem 的参考实现是中央搜索弹层，支持输入聚焦、Esc 关闭、方向键选择、Enter 选择和 Ctrl+1..9 快速选择。
- mXterm UI 必须继续使用 Radix、Lucide、全局 `--mx-*` token 和 `src/styles/app.css` 共享视觉体系。

## Requirements

- 在连接区提供一个搜索图标入口，打开中央连接快速搜索弹层，不在左侧连接树常驻输入框。
- 搜索范围包含连接名称、主机、端口、用户名、分组、备注、远端系统名称和版本。
- 空查询时显示最近连接优先的连接列表；有查询时按匹配相关性排序。
- 支持键盘操作：Esc 关闭、上下方向键切换、Enter 打开当前结果、Ctrl/Command+1..9 快速打开前九项。
- 搜索结果展示连接名称、地址、分组/系统等辅助信息，并标识当前活动连接或收藏状态。
- 弹层样式保持 mXterm 桌面工具风格，使用全局 token，不引入独立配色、原生下拉或新 UI 框架。
- 搜索匹配和排序逻辑需要抽成可验证的纯函数，避免散落在组件中。

## Acceptance Criteria

- [ ] 点击连接树工具栏搜索按钮能打开弹层，输入框自动聚焦。
- [ ] 连接可按名称、host、用户名、端口、分组、备注和远端系统字段搜索。
- [ ] 输入多个关键字时需要全部命中，避免单词过宽泛导致结果太多。
- [ ] 方向键、Enter、Esc、Ctrl/Command+1..9 行为可用。
- [ ] 选择结果后关闭弹层并调用现有连接打开流程。
- [ ] 无结果时有紧凑空状态，不遮挡或改变左侧连接树布局。
- [ ] 样式在浅色/深色主题下使用 `--mx-*` token，并复用项目弹层/按钮/图标节奏。
- [ ] `npm run check` 通过；新增的搜索 source-check 脚本通过。

## Out of Scope

- 不做终端输出搜索，本次只处理连接快速搜索。
- 不做全局命令面板或多资源搜索。
- 不改变连接数据结构和后端持久化格式。
- 不新增快捷键配置页。
