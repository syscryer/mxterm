# SSH 端口转发 / 隧道

## Goal

为 mXterm 增加 SSH 端口转发 / 隧道能力，让用户可以基于已有连接资料快速创建、启动、停止和查看 SSH 本地端口转发，用于访问只在远端网络可达的数据库、Web 服务或内网 API。

## User Value

- 不需要额外打开命令行执行 `ssh -L/-R/-D`。
- 隧道和现有 SSH 连接、跳板机、代理、凭据体系复用，减少重复配置。
- 可以在 mXterm 内看到隧道是否运行、绑定端口、目标地址和错误原因。

## Confirmed Facts

- 当前后端使用 `russh` 管理 SSH shell 和 SFTP，`TerminalSession` 已能解析保存连接、凭据、代理和 SSH 跳板机配置。
- 跳板机链路已使用 `channel_open_direct_tcpip` 打开 direct-tcpip 通道，说明后端具备实现本地转发的基础能力。
- 当前 `TerminalManager` 只管理 SSH 终端会话和本地终端会话；端口转发应有独立 manager，避免隧道生命周期绑死在 xterm tab 上。
- 前端工作区右侧工具栏当前有 `文件 / 传输 / 监控` 三个工具标签，右侧栏默认 360px、可拖拽调整，适合承载“隧道”工具页。
- Command Sender 已作为终端区底部面板存在，端口转发不适合放到该底部操作台，以免和命令发送场景混在一起。
- 连接资料已有 `proxy`、`jump`、`advanced`、凭据模式和远端系统字段；端口转发应复用这些连接资料，不重复保存密码或私钥内容。
- WindTerm README 声明支持本地、远程、动态三类端口转发；本地源码可确认其依赖 libssh 的 forwarding 能力，但未在当前开源仓库中定位到完整 UI/配置层。
- ui-ux-pro-max 对该类运维工具的建议是“数据密集但可扫描、状态颜色清晰、操作反馈明确”；实际落地必须贴合 mXterm 现有浅色/深色 token，不引入独立的黑绿终端风格。

## Requirements

- 提供一个“隧道”入口，优先放在右侧工具栏，与文件、传输、监控并列。
- 隧道规则持久化保存，列表展示名称、类型、所属连接、绑定地址、目标地址、自动启动开关、状态、错误信息和操作按钮。
- MVP 只支持本地端口转发：`local_host:local_port -> remote_host:remote_port`，通过选定 SSH 连接建立。
- 数据模型预留转发类型字段，便于后续扩展远程转发和动态 SOCKS，但 UI 第一版只开放本地转发。
- 启动隧道时复用保存连接的认证、代理、跳板机、known_hosts 校验和超时配置。
- 手动启动遇到 prompt 凭据连接时，前端弹出本次凭据输入；凭据只用于本次启动，不写入隧道规则或连接资料。
- 自动启动遇到 prompt 凭据连接时不弹窗，直接跳过该规则并显示“需要手动输入凭据”状态，避免应用启动时被凭据弹窗打断。
- 隧道运行状态必须可见：未启动、启动中、运行中、失败、已停止、需要凭据。
- 支持停止运行中的隧道；关闭 mXterm 时后端必须释放监听端口和 SSH 连接。
- 每条隧道规则可选择是否自动启动；默认关闭，应用启动后仅自动启动已显式开启的规则。
- 失败时展示可恢复的错误原因，例如本地端口占用、SSH 认证失败、known_hosts 需要确认、目标连接失败。
- 端口转发状态不应声称远端服务一定可用，只能表达“本地监听已建立”或“连接尝试失败”。
- UI 必须复用现有 Radix、Lucide、全局 `--mx-*` token 和共享样式，不使用原生下拉，不引入新 UI 框架。

## Acceptance Criteria

- [ ] 用户能在右侧工具栏进入“隧道”工具页。
- [ ] 用户能创建本地转发规则，选择已有 SSH 连接，填写本地监听地址/端口、远端目标地址/端口，并选择是否自动启动。
- [ ] 用户能启动、停止单条隧道，并看到状态变化和错误信息。
- [ ] 后端隧道生命周期独立于终端 tab；没有打开终端 tab 时也能通过保存连接启动隧道。
- [ ] 手动启动 prompt 凭据连接时会要求输入本次凭据，输入后可以继续启动隧道。
- [ ] 自动启动 prompt 凭据连接时不会弹窗，规则状态显示需要凭据。
- [ ] 后端启动失败时返回结构化 AppError，前端展示中文可理解错误。
- [ ] 本地端口占用、无效端口、缺少目标地址、连接缺失等输入错误被明确校验。
- [ ] `npm run check` 通过；后端新增 Rust 测试或 source-check 覆盖关键校验逻辑。

## Out of Scope For MVP

- 暂不做远程端口转发 `ssh -R`。
- 暂不做动态 SOCKS `ssh -D`。
- 暂不做全局自动恢复；只对显式开启“自动启动”的规则在应用启动后尝试恢复。
- 暂不做完整流量统计，只做基础状态和可选连接计数。
- 暂不把隧道规则写入连接资料本体，先作为独立规则/运行状态管理。
