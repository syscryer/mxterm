# 启动速度优化

## Goal

提升 mXterm 打包版启动速度和首屏响应，减少启动时的小窗口/空白等待、主题与连接信息加载前的卡顿，以及设置页相关重模块对主窗口首屏的影响。

## Requirements

- 优化必须优先基于证据定位启动路径上的同步工作、初始 bundle 依赖和非首屏重模块。
- 主窗口启动时应保持已实现的隐藏窗口策略：先应用启动主题和记忆窗口状态，再显示窗口。
- 非主窗口视图（如 VNC runner）不应在主窗口启动时被静态解析；主窗口不应在 VNC runner 启动时解析完整工作区。
- 设置、Docker、VNC、远程文件等非首屏/按需面板应尽量离开主窗口初始模块图；不得破坏已打开终端、远程文件、VNC 会话的挂载和状态保留。
- 优化不得引入生产环境噪声日志或明显 UI 闪烁。
- 不提交临时日志、诊断文件或敏感配置。

## Acceptance Criteria

- [ ] `main.tsx` 和 `App` 入口保持轻量启动，并按窗口视图加载对应应用模块。
- [ ] `WorkspaceShell` 不再静态导入明显非首屏的重型面板模块，按需渲染时动态加载。
- [ ] 终端配色数据预热仍在 idle/延后阶段执行，不阻塞首屏。
- [ ] `npm run check` 通过。
- [ ] `npm run build` 可完成，并能从构建输出看到重型模块进入独立 chunk 或离开初始 chunk。

## Notes

- 当前用户关注的是 release/打包版启动慢，dev 模式相对丝滑。
- 本任务先做前端启动模块图与首屏初始化优化；如果后续证据显示是 Rust/plugin 初始化或磁盘存储慢，再单独拆任务。
