# 优化新增分组界面

## Goal

优化左侧连接仓库中的“新增分组”交互，让创建态与树形列表风格一致，减少突兀感和表单感。

## Requirements

- 新增分组界面需要与左侧树节点的密度、圆角、边框和按钮语言保持一致。
- 保留分组名称输入、颜色选择、取消和保存能力。
- 新增和编辑分组改为弹出层，不再内嵌在左侧树列表中。
- 弹出层基于当前已接入的 Radix Dialog 实现，不新增不必要的 UI 框架。
- 删除连接、删除分组等确认操作不能再使用浏览器原生 `window.confirm`。
- 删除确认需要使用统一的自绘确认弹窗，风格接近现有连接编辑弹窗。
- 全局表单控件聚焦态需要移除浏览器默认黑色 outline，改为与项目主色一致的可见焦点样式。
- 无终端会话时，默认页需要跟随亮色主题，呈现为连接仓库/连接列表工作台，并作为可返回的首页入口；不再单独展示顶部协议分类条，标题栏左侧不展示刷新连接、新建分组、新建连接等重复图标工具，左侧树形区不再单独展示“全部”标题行，“分组”标题改为“连接”，并在该分区标题行参考 Codem 放置轻量上下文工具按钮（返回首页、刷新、更多、新增分组）；整体图标风格需要接近 Codem 的轻线性风格，默认图标颜色克制、hover 底色轻，系统图标保留识别性但外框和底色不能过重；筛选和分类统一收敛在列表工具栏中，工具栏不再单独显示“连接列表”标题，不使用“待命”等空泛标题；连接列表默认筛选为最近连接，筛选顺序为“最近 / 全部 / 收藏”，搜索提示使用真实字段表达；连接列表优先展示系统图标、最后连接、延迟、名称、地址、备注、分组和操作，不展示低价值的 CPU/内存/硬盘信息列；操作列在横向滚动时需要固定在右侧，但内容不能贴边，需要有明确内边距；行内操作不能通过整行 `button` 包裹多个操作按钮；系统列使用真实系统/发行版图标风格，不使用字母占位；新建 SSH 可作为主入口，但视觉需要克制，不使用厚重绿色按钮；导入连接归入“仓库维护”等低频维护入口，不在顶部工具栏重复展示；新建分组不在默认页右侧重复展示；文件、搜索、传输、监控等右侧会话工具跟随活动会话出现，默认页不展示。
- 弹出层背景需要有半透明遮罩和背景虚化效果。
- 鼠标点击弹出层外部背景时不能关闭弹窗，关闭必须通过关闭、取消或确认动作完成。
- 弹窗底部按钮需要有明确的主次层级和更精致的悬浮/禁用状态。
- 弹窗视觉需要保持简洁，圆角、阴影和按钮样式统一克制，避免过度渐变和厚重按钮。
- 无会话默认页的视觉继续参考 Codem：工具图标使用轻线性风格和淡 hover 状态，系统 logo 使用真实系统图标但以低饱和、淡外框的小徽标呈现；表格固定操作列只用轻分隔和足够内边距，避免厚重冻结面板感或贴边。

## Acceptance Criteria

- [ ] 新增/编辑分组通过弹出层完成，左侧树中不再出现内嵌创建表单。
- [ ] 删除连接、删除分组、连接编辑弹窗里的删除操作都使用自绘确认弹窗。
- [ ] 代码复用当前已接入的 Radix Dialog / Tooltip / ContextMenu，不引入多余 UI 框架。
- [ ] 输入框、下拉框、文本域聚焦时不再出现默认黑框，并仍保留清晰的键盘焦点提示。
- [ ] 提供无会话默认页 HTML 原型，覆盖亮色主题连接列表工作台；默认页作为首页入口展示，不展示右侧会话工具面板、顶部协议分类条、标题栏左侧重复图标工具和低价值主机信息列，连接列表默认选中“最近”，筛选顺序为“最近 / 全部 / 收藏”，最后连接列前置，操作列固定在右侧，备注列可以展示，系统列使用图标，顶部只保留新建 SSH 主入口且按钮视觉克制，低频导入入口不能抢占主视觉，新建分组不在右侧维护区重复展示，标题文案避免“待命”“次级操作”等生硬表达。
- [ ] 弹出层遮罩带背景虚化，且点击遮罩不会关闭弹窗。
- [ ] 弹窗按钮视觉层级清楚，主按钮、普通按钮、危险按钮有统一的美化状态。
- [ ] 弹窗圆角裁切正确，头部和底部区域不能破坏外层圆角。
