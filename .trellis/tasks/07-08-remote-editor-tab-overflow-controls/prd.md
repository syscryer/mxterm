# 编辑器标签栏溢出滚动控制

## Goal

远程编辑器标签数量过多时，标签栏不能继续挤压右侧工具入口或让最后一个标签被硬截断。标签栏需要提供类似 VSCode 的左右滚动控制，让用户可以在有限宽度内浏览全部已打开文件，并保持当前活动标签可见。

## Requirements

- 标签栏超出可视宽度时，显示或启用左右箭头控制。
- 点击左/右箭头应横向滚动标签列表，滚动距离接近一个标签宽度或可视区域的一部分。
- 没有可滚动内容时，箭头置灰或不可用。
- 打开、切换或关闭标签后，当前活动标签应自动滚动到可见区域。
- 右侧“文件 / 监控 / 命令 / 工具 / AI”等入口不能被标签挤压或遮挡。
- 标签本身仍保持现有紧凑桌面风格、关闭按钮、悬浮/选中状态和文本省略。
- 改动应使用现有 Lucide 图标、全局 token 和 `src/styles/app.css` 样式，不引入新 UI 框架。

## Acceptance Criteria

- [ ] 标签数量很多时，标签栏内部横向滚动，右侧工具入口位置稳定。
- [ ] 左右箭头类似 VSCode：可滚动时可点，不可滚动时置灰。
- [ ] 切换到被遮挡的标签时，该标签自动进入可视区域。
- [ ] 关闭活动标签或窗口宽度变化后，滚动状态能重新计算。
- [ ] `npm run check` 通过。
- [ ] `node scripts/check-startup-module-boundary-source.mjs` 通过。

## Notes

- 这是轻量 UI 修复，PRD-only 即可。
- 优先不做下拉溢出菜单，先实现 VSCode 式左右滚动。
