## 文档状态

## 1. 产品定位

mXterm 是一个面向个人使用的桌面运维客户端，用于集中管理服务器连接，并提供流畅稳定的 SSH 终端、SFTP 文件管理、上传下载队列和远程文本文件编辑能力。

首版目标是做出一个轻量、稳定、好用的个人连接管理工具，而不是完整复制 XTerminal 的全部协议、团队能力或企业审计能力。

## 2. 首版目标

1. 本地管理 SSH 连接，支持分组、收藏、最近连接和搜索。
2. 打开 SSH 终端会话，支持多会话、多标签和基础终端配置。
3. 通过 SFTP 浏览远程文件，支持上传、下载、拖拽上传、删除、重命名和新建目录。
4. 提供上传下载队列，显示进度、速度、状态，并支持取消和失败重试。
5. 支持远程文本文件编辑，使用 Monaco Editor 打开、修改和保存远程文件。
6. 支持当前文件查找替换和远程目录搜索。
7. 支持 Host Key 校验，首次连接和指纹变化必须给用户明确提示。
8. 密码和私钥口令使用系统钥匙串保存，编辑连接时允许用户主动查看。
9. 应用运行时不使用 Node/Express 作为本地后端服务，核心能力由 Rust/Tauri 提供。

## 3. 非目标范围

首版不做以下能力：

- RDP、VNC、Telnet。
- 团队用户、角色权限、共享连接仓库。
- 云端同步、云端仓库。
- 连接配置导入和导出。
- 堡垒机审计、录像、命令回放。
- 终端输入输出审计日志。
- 自动化批量执行命令。
- 插件市场或第三方扩展系统。
- Git 集成、LSP、代码补全、多人协作编辑。

这些能力可以作为后续版本规划，但首版不纳入实现范围。

## 4. 技术选型和技术红线

- 桌面框架：Tauri v2。
- 前端框架：React + TypeScript。
- 后端核心：Rust。
- SSH/SFTP 主线：`russh` + `russh-sftp`。
- 终端渲染：xterm.js。
- 远程文件编辑：Monaco Editor。
- 本地非敏感数据：SQLite。
- 敏感凭据：系统钥匙串，建议使用 `keyring` crate。
- 开发期构建工具：Node、npm、pnpm 和 `node_modules` 可以存在。
- 运行时红线：不使用 Node/Express 作为本地后端服务，不使用 Node SSH/SFTP 能力，不通过本地 Node 服务承载业务能力。

首版以 Windows 开发和验收为主，架构按 Windows、macOS、Linux 三端设计。平台差异集中在 Rust 的 platform 层，不散落到业务代码里。

## 5. 核心架构原则

1. 前端负责渲染、用户输入、轻量 UI 状态和状态展示。
2. Rust 后端负责 SSH、SFTP、传输队列、SQLite、系统钥匙串、Host Key 校验、跨平台路径和错误归一。
3. 前端通过 Tauri command 发起动作，Rust 通过 Tauri event 推送终端输出、连接状态、文件结果和传输进度。
4. SSH 终端、SFTP 浏览和传输队列不共用同一个底层 SSH 连接，避免文件传输影响终端手感。
5. 终端输出必须批量推送给前端，不按单字符高频发送事件。
6. SFTP、传输、远程编辑不能阻塞终端输入输出。

## 6. 功能需求

### 6.1 连接管理

连接管理用于维护本地 SSH 主机信息。

需要支持：

- 新增、编辑、删除连接。
- 连接分组。
- 收藏连接。
- 最近连接。
- 搜索连接。
- 连接备注。
- 标签。
- Host Key 首次信任和指纹变化确认。
- 测试连接。

连接基础字段：

- 连接名称。
- 主机地址。
- 端口。
- 用户名。
- 认证方式。
- 分组。
- 标签。
- 备注。
- 是否收藏。
- 终端配色覆盖项。

认证方式首版支持：

- 密码认证。
- 私钥认证。
- 私钥口令。

密码和私钥口令不得存入 SQLite 明文字段，必须存入系统钥匙串，并在 SQLite 中只保存引用 key。编辑连接时，如果系统钥匙串可以读取保存值，密码和私钥口令允许通过眼睛按钮显示。

### 6.2 SSH 终端

SSH 终端用于连接远程主机并进行命令交互。

需要支持：

- 打开连接。
- 关闭连接。
- 多终端标签。
- 一个连接下打开多个终端。
- 终端输入输出。
- 终端窗口大小变化同步。
- 复制和粘贴。
- 连接状态展示。
- 连接失败原因展示。
- 断线后保留终端 tab 和历史输出。
- 手动重连。
- 基础终端配置。

终端体验验收目标：

- 打字回显顺畅。
- `vim`、`top`、`htop`、`tail -f` 可用。
- 大量输出不导致 UI 卡死。
- SFTP 浏览和上传下载不影响终端输入手感。
- 用户主动关闭 tab 时不自动重连。
- 异常断开后显示断开状态，重连由用户手动触发。

基础终端配置包括：

- 字体大小。
- 字体族。
- 光标样式。
- 终端配色方案。
- 滚动缓冲行数。
- 复制和粘贴快捷键策略。

### 6.3 SFTP 文件管理

SFTP 文件管理用于浏览和操作远程文件系统。

需要支持：

- 打开远程目录。
- 路径跳转。
- 返回上级目录。
- 刷新目录。
- 文件排序。
- 新建目录。
- 删除文件或目录。
- 重命名文件或目录。
- 上传文件。
- 上传目录。
- 从本地文件管理器拖拽文件上传。
- 从本地文件管理器拖拽目录上传。
- 拖拽到当前目录空白区域时上传到当前远程路径。
- 拖拽到远程目录行时上传到该目录。
- 下载文件。
- 下载目录。
- 覆盖确认。
- 权限错误提示。

上传和下载必须通过统一传输队列管理。文件浏览使用独立 SFTP 连接，传输任务使用独立传输 worker 连接池，默认并发数为 2。

### 6.4 传输队列

传输队列用于管理所有上传和下载任务。

需要支持：

- 任务列表。
- 当前进度。
- 传输速度。
- 已传输大小。
- 总大小。
- 任务状态。
- 取消任务。
- 失败重试。
- 失败原因展示。
- 清空已完成任务。
- 打开本地保存位置。
- 跳转远程文件所在目录。

任务状态包括：

- 等待中。
- 传输中。
- 已完成。
- 已取消。
- 失败。

所有传输任务都在右侧工具面板的传输 tab 中展示。失败任务不自动消失，必须保留重试和移除入口。

### 6.5 远程文件编辑

远程文件编辑用于从 SFTP 文件列表中打开远程文本文件，并在客户端内编辑。

需要支持：

- 从 SFTP 文件列表打开文本文件。
- 使用 Monaco Editor 编辑文件内容。
- 多文件编辑标签。
- 修改后显示未保存标记。
- `Ctrl+S` 保存文件到远程原路径。
- 保存前检查远程文件是否发生变化。
- 保存失败时展示原因。
- 关闭未保存文件前提示。
- 当前文件 `Ctrl+F` 查找。
- 当前文件 `Ctrl+H` 替换。
- 当前文件查找支持大小写、全词和正则。
- 右侧搜索工具面板支持远程目录搜索。
- 搜索结果点击后打开文件并定位行号。

首版只支持文本文件编辑。

默认限制：

- 单文件编辑大小默认不超过 5 MB。
- 默认按 UTF-8 文本处理。
- 检测到二进制文件时禁止打开编辑。

保存前必须检查远程文件修改时间和文件大小。如果远程文件已经变化，不允许静默覆盖，必须提示用户选择重新加载、取消或确认覆盖。

当前文件替换只修改编辑器 buffer，不自动保存远程文件。用户必须通过 `Ctrl+S` 或保存按钮写回远程。

### 6.6 设置

设置入口固定在左下角。首版设置分为：

- 终端。
- 编辑器。
- 文件传输。
- 连接。
- 外观。
- 安全。

终端设置包括：

- 字体族。
- 字体大小。
- 光标样式。
- 滚动缓冲行数。
- 默认终端配色方案。
- 终端 tab 临时配色切换。

终端配色方案支持内置多套主题，并按以下优先级生效：

1. 当前终端临时选择。
2. 连接配置中的终端配色。
3. 全局默认终端配色。
4. mXterm 默认配色。

外观设置不叫主题。外观负责应用整体密度、字体大小、图标大小、面板尺寸记忆和启动时恢复上次布局。终端配色属于终端设置。

安全设置首版只保留结构。后续软件锁屏密码用于保护应用入口，支持启动时解锁、空闲锁定和休眠后锁定。

## 7. 本地数据与凭据

### 7.1 SQLite 数据

SQLite 保存非敏感配置。

建议表：

- `connections`：连接信息。
- `connection_groups`：连接分组。
- `connection_tags`：标签。
- `connection_tag_relations`：连接和标签关系。
- `recent_connections`：最近连接。
- `known_hosts`：Host Key 信任记录。
- `terminal_themes`：终端配色方案。
- `app_settings`：本地设置。
- `transfer_tasks`：传输任务历史。
- `editor_sessions`：编辑会话元信息。

SQLite 中禁止保存：

- 明文密码。
- 明文私钥口令。
- 未加密的敏感 token。

连接表需要保存系统钥匙串引用 key，例如：

- `password_secret_key`
- `private_key_passphrase_secret_key`

### 7.2 系统钥匙串

系统钥匙串保存敏感凭据。

需要保存：

- SSH 密码。
- 私钥口令。

服务名统一使用 `mxterm`。建议 key 规则：

- `connection.<connection_id>.password`
- `connection.<connection_id>.private_key_passphrase`

读取凭据失败时，应明确提示用户重新输入或重新保存。删除连接时，应先删除系统钥匙串中的密码和私钥口令，再删除 SQLite 连接记录。钥匙串删除失败时必须提示用户。

### 7.3 Host Key

`known_hosts` 表需要保存：

- 主机。
- 端口。
- key 类型。
- 指纹。
- 原始公钥。
- 首次信任时间。
- 最近使用时间。

未知主机首次连接必须提示用户选择取消、仅本次连接或信任并保存。指纹变化时必须强提示，不能自动覆盖。

## 8. Rust 后端模块

建议拆分为以下模块：

- `connection`：连接配置管理。
- `credential`：系统钥匙串读写。
- `storage`：SQLite 数据访问和迁移。
- `host_key`：Host Key 校验和信任记录。
- `terminal`：SSH 会话和终端输入输出。
- `sftp`：远程文件操作。
- `transfer`：上传下载队列和进度事件。
- `editor`：远程文本文件读取、保存和冲突检查。
- `settings`：本地设置。
- `theme`：终端配色方案。
- `platform`：跨平台路径、应用数据目录和系统能力。
- `error`：简单错误包装和脱敏。
- `event`：统一事件回推。

后端通过 Tauri command 暴露能力给前端，通过事件机制回推长任务状态。

## 11. 错误处理

首版采用简单错误包装，不做复杂错误中心。

错误结构需要包含：

- `code`：归一化错误码。
- `message`：用户可读提示。
- `raw_message`：底层原始错误详情。
- `recoverable`：是否可重试或可修复。

界面默认显示 `message`，允许展开或复制 `raw_message`。原始错误需要做基本脱敏，避免包含密码和私钥口令。

需要明确处理以下错误：

- SSH 连接超时。
- SSH 认证失败。
- Host Key 未知。
- Host Key 变化。
- 网络中断。
- SFTP 权限不足。
- 文件不存在。
- 文件上传覆盖冲突。
- 文件下载本地路径不可写。
- 远程文件编辑保存冲突。
- 系统钥匙串读取失败。
- SQLite 读写失败。

## 12. 安全要求

1. 不做终端输入输出审计。
2. 不记录明文密码。
3. 不在日志中输出敏感凭据。
4. SQLite 不保存明文密码和私钥口令。
5. 系统钥匙串读取失败时，不做静默降级。
6. 编辑连接时允许用户主动查看已保存密码和私钥口令。
7. SSH Host Key 需要校验，首次连接和指纹变化必须提示用户确认。
8. 连接配置导出不在首版实现。后续实现时默认不导出敏感凭据。

## 13. 性能要求

1. 终端输出不能阻塞界面。
2. 终端输出通过批量事件推送给前端。
3. 上传下载使用流式处理，避免一次性读取大文件。
4. 大目录加载需要支持加载状态。
5. 文件编辑默认限制 5 MB，避免 Monaco 打开超大文件卡顿。
6. 传输队列默认并发数为 2，可在设置中调整。
7. SFTP 浏览和传输队列不能影响终端输入响应。

## 15. 验收标准

首版完成时，应满足：

1. 可以新增一个 SSH 连接并保存。
2. 可以查看编辑连接中的已保存密码或私钥口令。
3. 可以完成 Host Key 首次信任。
4. Host Key 变化时会阻止静默连接并提示用户。
5. 可以使用保存的连接打开 SSH 终端。
6. 可以同时打开多个连接标签。
7. 可以在一个连接下打开多个终端。
8. 终端输入回显顺畅，大量输出不会卡死 UI。
9. 可以浏览远程目录。
10. 可以上传文件到远程目录。
11. 可以拖拽文件或目录上传到远程目录。
12. 可以从远程目录下载文件。
13. 可以查看上传下载进度。
14. 可以取消失败或进行中的传输任务。
15. 可以打开远程文本文件并编辑。
16. 可以使用当前文件查找和替换。
17. 可以保存远程文本文件。
18. 保存远程文件时能检测远程变更冲突。
19. 可以执行远程目录搜索并打开搜索结果。
20. 可以选择终端配色方案。
21. 可以调整基础外观设置。
22. 密码和私钥口令不以明文形式保存到 SQLite。
23. 连接失败、认证失败、权限不足等错误有明确提示和原始详情。
24. 发布版不依赖 Node/Express 运行时后端。

## 16. MVP 开发顺序

### 阶段一：工程基座

- 初始化 Tauri + React + TypeScript。
- 建立 Rust 模块结构。
- 建立 Trellis `dev`、`check`、`test` 工作流。
- 迁入当前原型主布局。
- 明确运行时不使用 Node/Express 本地后端。

### 阶段二：SSH 体验 Spike
