## 1. 产品定位

mXterm 是一个面向个人使用的桌面运维客户端，用于集中管理服务器连接，并提供 SSH 终端、SFTP 文件管理、上传下载队列和远程文本文件编辑能力。

首版目标是做出一个轻量、稳定、好用的个人连接管理工具，而不是完整复制 XTerminal 的全部协议和团队能力。

## 2. 首版目标

1. 本地管理 SSH 连接，支持分组、收藏、最近连接和搜索。
2. 打开 SSH 终端会话，支持多会话、多标签和基础终端配置。
3. 通过 SFTP 浏览远程文件，支持上传、下载、删除、重命名和新建目录。
4. 提供上传下载队列，显示进度、速度、状态，并支持取消和失败重试。
5. 支持远程文本文件编辑，使用内置 Monaco Editor 打开、修改和保存远程文件。
6. 敏感凭据使用系统钥匙串保存，避免明文落盘。

## 3. 非目标范围

首版不做以下能力：

- RDP、VNC、Telnet。
- 团队用户、角色权限、共享连接仓库。
- 云端同步、云端仓库。
- 堡垒机审计、录像、命令回放。
- 自动化批量执行命令。
- 插件市场或第三方扩展系统。

这些能力可以作为后续版本规划，但首版不纳入实现范围。

## 4. 技术选型

- 桌面框架：Tauri。
- 前端框架：React + TypeScript。
- 后端核心：Rust。
- 终端渲染：xterm.js。
- 远程文件编辑：Monaco Editor。
- 本地非敏感数据：SQLite。
- 敏感凭据：系统钥匙串。

## 5. 功能需求

### 5.1 连接管理

连接管理用于维护本地 SSH 主机信息。

需要支持：

- 新增、编辑、删除连接。
- 连接分组。
- 收藏连接。
- 最近连接。
- 搜索连接。
- 连接备注。
- 标签。
- 导入和导出连接配置。

连接基础字段：

- 连接名称。
- 主机地址。
- 端口。
- 用户名。
- 认证方式。
- 分组。
- 标签。
- 备注。
- 是否收藏。

认证方式首版支持：

- 密码认证。
- 私钥认证。
- 私钥口令。

密码和私钥口令不得存入 SQLite 明文字段，必须存入系统钥匙串，并在 SQLite 中只保存凭据引用。

### 5.2 SSH 终端

SSH 终端用于连接远程主机并进行命令交互。

需要支持：

- 打开连接。
- 关闭连接。
- 多终端标签。
- 终端输入输出。
- 终端窗口大小变化同步。
- 复制和粘贴。
- 连接状态展示。
- 连接失败原因展示。
- 手动重连。
- 基础终端配置。

基础终端配置包括：

- 字体大小。
- 字体族。
- 光标样式。
- 主题。
- 滚动缓冲行数。

### 5.3 SFTP 文件管理

SFTP 文件管理用于浏览和操作远程文件系统。

需要支持：

- 打开远程目录。
- 路径跳转。
- 返回上级目录。
- 刷新目录。
- 文件排序。
- 新建目录。
- 删除文件或目录。
- 重命名文件或目录。
- 上传文件。
- 上传目录。
- 下载文件。
- 下载目录。
- 覆盖确认。
- 权限错误提示。

上传和下载必须通过统一传输队列管理。

### 5.4 传输队列

传输队列用于管理所有上传和下载任务。

需要支持：

- 任务列表。
- 当前进度。
- 传输速度。
- 已传输大小。
- 总大小。
- 任务状态。
- 取消任务。
- 失败重试。
- 失败原因展示。

任务状态包括：

- 等待中。
- 传输中。
- 已完成。
- 已取消。
- 失败。

首版可以限制并发传输数量，默认并发数为 2。

### 5.5 远程文件编辑

远程文件编辑用于从 SFTP 文件列表中打开远程文本文件，并在客户端内编辑。

需要支持：

- 从 SFTP 文件列表打开文本文件。
- 使用 Monaco Editor 编辑文件内容。
- 多文件编辑标签。
- 保存文件到远程原路径。
- 保存前检查远程文件是否发生变化。
- 保存失败时展示原因。
- 关闭未保存文件前提示。

首版只支持文本文件编辑。

默认限制：

- 单文件编辑大小默认不超过 5 MB。
- 默认按 UTF-8 文本处理。
- 检测到二进制文件时禁止打开编辑。

远程文件保存前必须检查：

- 远程文件修改时间。
- 远程文件大小。

如果远程文件已经变化，不允许静默覆盖，必须提示用户选择重新加载或确认覆盖。

## 6. 本地数据与凭据

### 6.1 SQLite 数据

SQLite 保存非敏感配置。

建议表：

- `connections`：连接信息。
- `connection_groups`：连接分组。
- `connection_tags`：标签。
- `connection_tag_relations`：连接和标签关系。
- `recent_connections`：最近连接。
- `settings`：本地设置。
- `transfer_history`：传输历史摘要。

SQLite 中禁止保存：

- 明文密码。
- 明文私钥口令。
- 未加密的敏感 token。

### 6.2 系统钥匙串

系统钥匙串保存敏感凭据。

需要保存：

- SSH 密码。
- 私钥口令。

SQLite 中只保存凭据引用，例如：

- 凭据类型。
- 凭据 key。
- 更新时间。

读取凭据失败时，应明确提示用户重新输入或重新保存凭据。

## 7. Rust 后端模块

建议拆分为以下模块：

- `connection`：连接配置管理。
- `credential`：系统钥匙串读写。
- `storage`：SQLite 数据访问和迁移。
- `terminal`：SSH 会话和终端输入输出。
- `sftp`：远程文件操作。
- `transfer`：上传下载队列和进度事件。
- `editor`：远程文本文件读取、保存和冲突检查。
- `settings`：本地设置。
- `event`：统一事件回推。

后端通过 Tauri command 暴露能力给前端，通过事件机制回推长任务状态。

## 10. 错误处理

需要明确处理以下错误：

- SSH 连接超时。
- SSH 认证失败。
- 主机密钥校验失败。
- 网络中断。
- SFTP 权限不足。
- 文件不存在。
- 文件上传覆盖冲突。
- 文件下载本地路径不可写。
- 远程文件编辑保存冲突。
- 系统钥匙串读取失败。
- SQLite 读写失败。

错误提示必须面向用户说明可执行动作，例如重新输入凭据、检查权限、重新连接、选择其他保存路径。

## 11. 安全要求

1. 不记录明文密码。
2. 不在日志中输出敏感凭据。
3. SQLite 不保存明文密码和私钥口令。
4. 系统钥匙串读取失败时，不做静默降级。
5. 导出配置时，默认不导出敏感凭据。
6. 如果后续支持导出敏感凭据，必须使用用户输入的导出密码加密。
7. SSH 主机密钥需要校验，首次连接时应提示用户确认。

## 12. 性能要求

1. 终端输出不能阻塞界面。
2. 上传下载使用流式处理，避免一次性读取大文件。
3. 大目录加载需要支持加载状态。
4. 文件编辑默认限制 5 MB，避免 Monaco 打开超大文件卡顿。
5. 传输队列默认并发数为 2，可后续在设置中开放调整。

## 13. 给 Gemini 的界面设计输入

Gemini 负责界面设计时，需要覆盖以下页面和状态：

1. 连接管理首页。
2. 新增和编辑连接表单。
3. SSH 终端工作区。
4. SFTP 文件管理面板。
5. 上传下载队列。
6. Monaco 文件编辑标签页。
7. 凭据输入和保存提示。
8. 设置页。
9. 错误提示、覆盖确认、保存冲突确认。
10. 空状态、加载状态、失败状态。

界面设计需要体现这是个人桌面工具，不是营销网站。优先考虑高频运维效率、信息密度、可扫描性和重复操作的顺手程度。

## 14. 验收标准

首版完成时，应满足：

1. 可以新增一个 SSH 连接并保存。
2. 可以使用保存的连接打开 SSH 终端。
3. 可以同时打开多个终端标签。
4. 可以浏览远程目录。
5. 可以上传文件到远程目录。
6. 可以从远程目录下载文件。
7. 可以查看上传下载进度。
8. 可以取消失败或进行中的传输任务。
9. 可以打开远程文本文件并编辑。
10. 可以保存远程文本文件。
11. 保存远程文件时能检测远程变更冲突。
12. 密码和私钥口令不以明文形式保存到 SQLite。
13. 连接失败、认证失败、权限不足等错误有明确提示。
